# Что осталось сделать и как — по порядку

У вас уже есть: бот + мини-апп + оплата на сервере, инструкция по скачиванию Happ.  
Ниже — что осталось и как сделать **по шагам**.

---

## Общая схема (что за что отвечает)

```
[Пользователь] → оплачивает в мини-апп → [Бот]
       ↓
Бот вызывает API Happ → получает install_code
       ↓
Бот отправляет пользователю ссылку: HAPP_SUBSCRIPTION_URL?installid=КОД
       ↓
Пользователь вставляет ссылку в приложение Happ
       ↓
Приложение Happ запрашивает по этой ссылке список серверов
       ↓
Список серверов должен отдавать ВАШ сервер (3x-ui или аналог)
```

**Итого:** нужен сервер с панелью (3x-ui), который отдаёт подписку по ссылке. Эта ссылка и есть **HAPP_SUBSCRIPTION_URL**.

---

## Что осталось сделать (список)

1. Арендовать VPS (если ещё нет).
2. Установить на VPS панель 3x-ui и настроить один inbound (сервер VPN).
3. Взять из 3x-ui ссылку подписки — это будет базовая ссылка для Happ.
4. Зарегистрироваться как провайдер на Happ, взять provider_code и auth_key.
5. В кабинете Happ привязать домен подписки (если требуется по инструкции Happ).
6. В .env на сервере (где крутится бот) прописать HAPP_* переменные, в том числе HAPP_SUBSCRIPTION_URL.
7. Перезапустить бота и проверить оплату.

Дальше — пошагово.

---

## Шаг 1. VPS для 3x-ui

- Нужен отдельный сервер под VPN (можно тот же, где бот, но лучше отдельный VPS для изоляции).
- Минимум: 1 ядро CPU, 1 GB RAM, 15 GB диск, Ubuntu 20.04+ или Debian 11+.
- Регион: Германия или Нидерланды (как обсуждали).

Провайдеры: Hetzner, Contabo, и т.д. Создаёте VPS, подключаетесь по SSH (root или пользователь с sudo).

---

## Шаг 2. Установка 3x-ui на VPS

Подключаетесь к VPS по SSH и выполняете **одну команду**:

```bash
sudo bash <(curl -Ls https://raw.githubusercontent.com/MHSanaei/3x-ui/master/install.sh)
```

Скрипт спросит:
- порт панели (по умолчанию 2053 — можно оставить);
- имя пользователя и пароль панели — **обязательно смените** с дефолтного admin/admin.

После установки панель доступна по адресу:

- `http://IP_ВАШЕГО_VPS:2053`

Например: `http://123.45.67.89:2053`.  
Зайдите в панель и сразу смените пароль: **Panel Settings → Authentication**.

---

## Шаг 3. Настройка inbound и получение ссылки подписки

1. В меню 3x-ui выберите **Inbounds** (Входящие).
2. Нажмите **Add Inbound** (Добавить входящий).
3. Заполните:
   - **Remark** — любое название (например, "BIT VPN DE").
   - **Protocol** — например **VLESS** (рекомендуется) или VMess.
   - **Port** — порт для подключения клиентов (например, 443 или 8443). Не путать с портом панели 2053.
   - Остальное можно оставить по умолчанию или по документации 3x-ui.
4. Сохраните inbound.
5. В списке inbound откройте созданный и найдите **клиента** (client). У каждого клиента есть **Subscription link** (ссылка подписки). Она выглядит примерно так:
   - `http://IP:2053/sub/XXXXXXXX`
   - или с поддоменом: `https://sub.ваш-домен.com/sub/XXXXXXXX`

Эта ссылка — **ваша базовая ссылка подписки**. По ней приложение Happ будет запрашивать список серверов.  
Пока можно использовать вариант с IP: `http://IP_ВАШЕГО_VPS:2053/sub/XXXXXXXX`.  
Для продакшена лучше повесить домен и HTTPS (см. шаг 3.1).

### 3.1. Домен и HTTPS (желательно для продакшена)

- Купите домен или используйте поддомен (например, `sub.ваш-домен.ru`).
- Настройте A-запись домена на IP вашего VPS.
- В 3x-ui в настройках панели можно указать домен и включить TLS (или поставить Nginx/Caddy перед панелью с Let's Encrypt).  
Тогда ссылка подписки будет вида: `https://sub.ваш-домен.ru/sub/XXXXXXXX` — её и нужно использовать как базу для Happ.

---

## Шаг 4. Регистрация как провайдер на Happ

1. Зайдите на [happ-proxy.com](https://happ-proxy.com) (или [happ.su](https://www.happ.su/)).
2. Зарегистрируйтесь / войдите как **провайдер**.
3. В кабинете найдите:
   - **provider_code** (8 символов);
   - **auth_key** (ключ API, обычно 32 символа).

Сохраните их — понадобятся для .env.

---

## Шаг 5. Привязка домена подписки в Happ (если требуется)

По документации Happ для лимитированных ссылок домен подписки может потребоваться привязать к вашему provider через API:

- Эндпоинт: `GET https://happ-proxy.com/api/add-domain`
- Параметры: `provider_code`, `auth_key`, `domain_hash` (SHA-256 от домена в нижнем регистре).

Если вы пока используете ссылку по IP (`http://IP:2053/sub/...`), уточните в поддержке Happ, можно ли так или нужен именно домен. Если нужен домен — сначала настройте домен и HTTPS (шаг 3.1), затем привяжите домен через их API (примеры есть в [документации Happ](https://docs.happ-proxy.com)).

---

## Шаг 6. Прописать переменные в .env на сервере

На сервере, где запущен бот (например, `/opt/vpn-bot/`), откройте файл `.env` и добавьте/заполните:

```env
HAPP_API_URL=https://happ-proxy.com
HAPP_PROVIDER_CODE=ваш_8_символов
HAPP_AUTH_KEY=ваш_32_символа
HAPP_SUBSCRIPTION_URL=http://IP_ВАШЕГО_VPS:2053/sub/XXXXXXXX
```

- **HAPP_SUBSCRIPTION_URL** — это **точно** та ссылка подписки, которую вы скопировали из 3x-ui (шаг 3), **без** параметра `installid` (бот сам добавит `?installid=...` при выдаче пользователю).
- Если настроили домен и HTTPS — подставьте ссылку вида `https://sub.ваш-домен.ru/sub/XXXXXXXX`.

Сохраните файл.

---

## Шаг 7. Перезапуск бота и проверка

На сервере:

```bash
sudo systemctl restart vpn-bot
```

Проверка:

1. Сделайте тестовую оплату (или проверку платежа в боте).
2. После успешной оплаты пользователь должен получить сообщение со ссылкой и файл .txt.
3. Откройте эту ссылку в браузере или вставьте в приложение Happ — должно подтянуться подключение к вашему серверу из 3x-ui.

Если ссылка не приходит или Happ пишет ошибку — смотрите логи бота:

```bash
journalctl -u vpn-bot -n 100 --no-pager
```

Ошибки вызова API Happ там видны (например, "Happ API add-install failed").

## Краткий чек-лист

| № | Действие | Где/как |
|---|----------|--------|
| 1 | VPS (1 CPU, 1 GB RAM, Ubuntu 20+ / Debian 11+) | Hetzner, Contabo и т.д. |
| 2 | Установить 3x-ui | `sudo bash <(curl -Ls https://raw.githubusercontent.com/MHSanaei/3x-ui/master/install.sh)` |
| 3 | Зайти в панель, сменить пароль | http://IP_VPS:2053 → Panel Settings → Authentication |
| 4 | Создать Inbound (VLESS/VMess), взять ссылку подписки | Inbounds → Add → скопировать Subscription link |
| 5 | (Желательно) Домен + HTTPS для ссылки подписки | A-запись на VPS, TLS в панели или Nginx/Caddy |
| 6 | Провайдер на Happ, взять provider_code и auth_key | happ-proxy.com |
| 7 | Привязать домен в Happ (если нужен) | API add-domain по документации Happ |
| 8 | В .env бота прописать HAPP_* и HAPP_SUBSCRIPTION_URL | Ссылка из п.4 (базовая, без installid) |
| 9 | Перезапустить бота, проверить оплату | systemctl restart vpn-bot |

После этого цепочка «оплата → ссылка Happ → подключение через Happ к вашим серверам в 3x-ui» должна работать полностью.
