# Аудит проекта VPN BOT (Bit VPN)

**Последнее обновление:** март 2026 (полный проход: бот ↔ мини-апп, оплата, GitHub, безопасность).

---

## 1. Архитектура и синхронизация бот ↔ мини-апп

| Компонент | Роль |
|-----------|------|
| **Бот** (`bot/`, `run.py`) | Команды, оплата в чате, админка, выдача подписки, Happ. |
| **Mini App API** (`api_miniapp.py`) | FastAPI: `initData` (HMAC с `BOT_TOKEN`), `/me`, `/plans`, `/create-payment`, вебхук ЮKassa, редирект `/sub/...`. |
| **Фронт** (`webapp/index.html` → копии `public/`, `api/`) | Запросы к API по `?api=...` из `WEBAPP_URL` / `MINIAPP_API_URL`. |
| **База** | Общая `DATABASE_URL` (Neon и т.д.) — бот и API читают одних пользователей и подписки. |
| **Тарифы** | `PLAN_*_PRICE` в `.env` + `SUBSCRIPTION_PLANS` в `bot/config/settings.py` — должны совпадать на боте и на хостинге API (Vercel/сервер). |

Подробнее: **`docs/miniapp/СИНХРОНИЗАЦИЯ_БОТ_МИНИАПП.md`**.

**Чеклист перед продом:** один и тот же `BOT_TOKEN`, `DATABASE_URL`, цены `PLAN_*`, `MINIAPP_API_URL` в ссылке мини-аппа, после смены переменных — redeploy API и `restart` бота.

## 2. Безопасность (сделано в коде)

| Тема | Статус |
|------|--------|
| **Проверка `initData`** | HMAC-SHA256 по документации Telegram Web App. |
| **Вебхук ЮKassa** | После уведомления платёж **дополнительно проверяется** через API ЮKassa (`GET /payments/{id}`): статус `succeeded` и **совпадение суммы** с записью в БД. Сырые POST без проверки не принимаются. |
| **CORS** | По умолчанию можно задать **`MINIAPP_CORS_ORIGINS`** (список доменов через запятую). `*` оставлено для отладки. |
| **OpenAPI `/docs`** | По умолчанию **выключено** (`API_EXPOSE_DOCS=0`). Включать только при необходимости. |
| **Отладочные эндпоинты** (`/api/miniapp/debug-*`, `check-happ-env`) | Доступны **только при `MINIAPP_EXPOSE_DEBUG=1`**, иначе **404**. |
| **Лимит оплаты** | `POST /api/miniapp/create-payment`: до **`MINIAPP_CREATE_PAYMENT_RATE`** запросов в минуту с одного IP (по умолчанию 8). |
| **Секреты в Git** | `.env` в `.gitignore`. **`deploy_config.cmd`** с IP сервера — в `.gitignore`; в репозитории только **`deploy_config.example.cmd`**. |
| **Админка бота** | Доступ по `ADMIN_IDS` в `bot/handlers/admin.py` (через `is_admin`). |

## 3. GitHub — что не выкладывать

- Никогда: **`.env`**, реальный **`deploy_config.cmd`** (IP, привычные пути).
- В репозитории: **`.env.example`**, **`deploy_config.example.cmd`**.
- Если IP уже попал в историю Git — смените IP/хост или считайте его скомпрометированным для скрытности.

## 4. Оплата

- **ЮKassa:** создание платежа в `bot/utils/payments.py`, вебхук `POST /api/webhook/yookassa` в `api_miniapp.py` с **верификацией через API**.
- **Проверка в боте** «Проверить платёж» — по-прежнему через API ЮKassa.
- Настройки: `docs/payments/ЮKASSA_ИНТЕГРАЦИЯ.txt`, `docs/payments/ОПЛАТА_И_ССЫЛКИ_ПРОВЕРКА.md`.

## 5. Что ещё можно усилить (по желанию)

- Проверка **IP-адресов** отправителя вебхука ЮKassa (список в документации ЮKassa).
- **Request ID** в логах API.
- Автотесты `pytest` для критичных эндпоинтов.
- Отдельный секрет **`WEBHOOK_SECRET`** в URL вебхука (если поддерживается настройкой в личном кабинете).

## 6. Краткая оценка

| Категория | Оценка |
|-----------|--------|
| Связка бот / мини-апп / БД | Готово при совпадении env |
| Оплата и вебхук | Защита по API ЮKassa включена |
| Секреты и деплой | Шаблоны в репо, локальное в gitignore |
| Поверхность API | CORS, отключение docs, debug только по флагу, rate limit |

После смены VPS: обновить **DNS**, **SSL**, **`MINIAPP_API_URL` / `WEBAPP_URL`**, переменные на Vercel, **`HAPP_SUBSCRIPTION_REDIRECT_BASE`** при необходимости.
