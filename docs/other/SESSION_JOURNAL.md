# Журнал сессий с Cursor

Файл для сохранения контекста переписок и сделанных правок. При новых задачах агент читает этот файл и дополняет его после работы.

---

## Как пользоваться

- **Перед задачей:** агент смотрит последние записи, чтобы не повторять ошибки и учитывать уже сделанное.
- **После правок:** агент добавляет краткую запись: дата, что сделано, какие файлы затронуты, что проверить или что ещё может быть не так.

### Git: ветка main (локально и на GitHub)

**Запрос:** Запомнить и записать в журнал, чтобы не давать пользователю неверные инструкции (в т.ч. по git push).

**Контекст:**
- В репозитории VPN BOT везде используется ветка **main** (локально и на GitHub, в деплое указана main).
- Если `git push` выдаёт ошибку про upstream — значит upstream не задан или указывает на другую ветку.

**Правильные команды для push (указывать в инструкциях):**
- Отправить изменения в удалённую ветку main:
  ```cmd
  git push origin main
  ```
- Чтобы в дальнейшем достаточно было `git push`, один раз задать upstream:
  ```cmd
  git branch --set-upstream-to=origin/main main
  ```

**Правило для агента:** В инструкциях по деплою писать **`git push origin main`**. Не предлагать push в master — в этом проекте рабочая ветка main.

### Имя и описание подписки в Happ (как у LastdepVPN, без домена)

**Запрос:** Чтобы в Happ у всех отображалось только название VPN (без цифр), сервер — 95.181.175.67; плюс описание (поддержка, бот, тарифы). Домена нет.

**Сделано:**
- В контенте подписки при прокси через `/sub/КОД` подменяются поле **ps** (remark) в ссылках vmess/vless/trojan/ss на заданные имя и описание. Подключение остаётся к серверу из нод (95.181.175.67).
- Переменные в `.env`: **SUBSCRIPTION_DISPLAY_NAME** (по умолчанию "BIT VPN"), **SUBSCRIPTION_DESCRIPTION** (многострочно; в .env `\n` — новая строка).
