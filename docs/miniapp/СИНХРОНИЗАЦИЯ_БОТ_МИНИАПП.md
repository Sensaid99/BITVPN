# Синхронизация бота в Telegram и мини-приложения

Чтобы **цены, тарифы, профиль и настройки** были одинаковыми в боте и в мини-апп, всё берётся из одного источника: **.env** и код **bot/config/settings.py**. API мини-апп на Vercel при загрузке читает те же переменные окружения.

---

## 1. Единый источник правды

| Что | Где задаётся | Где используется |
|-----|----------------|-------------------|
| Цены тарифов (1/3/6/9/12 мес) | .env: `PLAN_1_MONTH_PRICE`, `PLAN_3_MONTH_PRICE`, … | Бот, API (мини-апп получает через `/api/miniapp/plans`) |
| Цены по устройствам (1, 3, 5, 10) | `bot/config/settings.py`: `DEVICE_BASE_PRICE` | Бот, API (в ответе `pricing.device_base_price`) |
| Имена тарифов, описание, emoji | `bot/config/settings.py`: `SUBSCRIPTION_PLANS` | Бот, API |
| Поддержка (username бота) | .env: `SUPPORT_USERNAME` | Бот, API → мини-апп (ссылка «Поддержка») |
| Реферальная программа | .env: `REFERRAL_BONUS_PERCENT`, `REFERRAL_MIN_PAYOUT` | Бот, API (в ответе `config`) |
| Профиль пользователя, подписки | База данных (Neon) | Бот, API (`/api/miniapp/me`) |

Мини-апп **ничего не хранит в себе** по тарифам и ценам — при открытии запрашивает `/api/miniapp/plans` и `/api/miniapp/me`, поэтому всегда показывает то, что отдаёт API (и бот).

---

## 2. Что прописать в Vercel (Environment Variables)

Чтобы на Vercel API отдавал **те же цены и настройки**, что и бот на ПК, в проекте API на Vercel добавьте те же переменные, что и в **.env** на ПК:

**Обязательно (без них API может не поднять конфиг или отдавать дефолтные цены):**
- `BOT_TOKEN`
- `DATABASE_URL`

**Для синхронизации тарифов и поддержки (скопируйте значения из .env):**
- `PLAN_1_MONTH_PRICE` (например 299)
- `PLAN_3_MONTH_PRICE` (например 799)
- `PLAN_6_MONTH_PRICE` (например 1499)
- `PLAN_9_MONTH_PRICE` (если есть)
- `PLAN_12_MONTH_PRICE` (например 2699)
- `SUPPORT_USERNAME` (например HelpBit_bot — без @)
- `REFERRAL_BONUS_PERCENT` (например 10)
- `REFERRAL_MIN_PAYOUT` (например 100)

После изменения переменных в Vercel нажмите **Redeploy**, чтобы применились новые значения.

---

## 3. Где что менять

- **Цены тарифов (руб за 1 устройство):** в **.env** на ПК (`PLAN_1_MONTH_PRICE`, …) и **те же значения** в Vercel → Environment Variables. Затем перезапуск бота и Redeploy API.
- **Цены по числу устройств (1, 3, 5, 10):** в **bot/config/settings.py**, константа `DEVICE_BASE_PRICE`. После правки — перезапуск бота и деплой API (чтобы на Vercel подхватился обновлённый код).
- **Названия тарифов, описание, emoji:** в **bot/config/settings.py**, словарь `SUBSCRIPTION_PLANS`. Аналогично: правка кода → перезапуск бота + деплой API.
- **Поддержка (username):** в **.env** — `SUPPORT_USERNAME`; то же значение в Vercel. Мини-апп возьмёт его из ответа API (`config.support_username`).
- **Профиль и подписки:** общая база (Neon). Меняются только через бота/оплату; и бот, и мини-апп читают одни и те же данные.

---

## 4. Проверка

1. В боте: тарифы, цены, кнопка поддержки.
2. Откройте мини-апп: те же тарифы и цены (в том числе при смене срока и устройств), та же ссылка на поддержку.
3. Если в мини-апп старые цены — проверьте, что в Vercel заданы `PLAN_*_PRICE` и сделан Redeploy.

---

## 5. Сайт (лендинг `website/index.html`) и тот же бэкенд

Лендинг в репозитории обращается к **`POST /api/site/register`** по **тому же origin**, что и страница (`fetch(window.location.origin + "/api/site/register", …)`). То есть в продакшене HTML и API должны быть **на одном домене** (например `bitecosystem.ru`), а **nginx** (или другой прокси) направляет:

- статику сайта — на файлы или CDN;
- пути **`/api/`** — на процесс **miniapp-api** (`api_miniapp.py`), где уже есть `POST /api/site/register`.

**База:** тот же **`DATABASE_URL`**, что у бота и у API. Регистрация с сайта пишет в ту же Neon/PostgreSQL, что и бот — дубликаты и логика уникальности email обрабатываются в коде эндпоинта.

**Проверка:** после деплоя откройте форму на сайте и убедитесь в логах **miniapp-api**, что запрос доходит без **404**; если **404** на `/api/site/register` — сайт открыт с другого хоста, чем API, или прокси не прокидывает `/api/`.

Мини-апп по-прежнему ходит к API по URL из **`?api=`** или настроенному хосту (см. **API_VERCEL_ПОШАГОВО.md**); с сайтом это не смешивается, пока не унифицируете домены осознанно.
