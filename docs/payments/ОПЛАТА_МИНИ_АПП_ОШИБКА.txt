  «Не удалось создать платёж» при любом способе оплаты в Mini App
═══════════════════════════════════════════════════════════════

  Если при нажатии ЮKassa, СБП или другого способа всегда одна и та же
  ошибка — запрос к API создания платежа не доходит до рабочего кода
  или API не может создать платёж (нет БД/ключей).

1. ПРОВЕРЬТЕ ПЕРЕМЕННЫЕ НА VERCEL

   Mini App открывается с bitvpn.vercel.app, запросы идут туда же.
   На Vercel в настройках проекта должны быть заданы переменные:

   • DATABASE_URL    — строка подключения к PostgreSQL (как в .env)
   • BOT_TOKEN       — токен бота
   • YOOKASSA_SHOP_ID
   • YOOKASSA_SECRET_KEY

   Без DATABASE_URL и BOT_TOKEN API не сможет проверить пользователя
   и создать платёж. Добавьте их в Vercel → Project → Settings →
   Environment Variables, затем сделайте Redeploy.

2. УБЕДИТЕСЬ, ЧТО В ДЕПЛОЕ ЕСТЬ ПАПКА BOT

   На Vercel деплоится репозиторий. В нём должна быть папка bot/
   (config, models, utils и т.д.). Если в vercel.json или .vercelignore
   исключена папка bot, API не сможет импортировать настройки и БД.

3. ВАРИАНТ: API НА ВАШЕМ СЕРВЕРЕ

   Если не хотите настраивать Vercel, можно направить запросы на свой
   сервер, где уже есть .env и база. Тогда в .env укажите:

     WEBAPP_URL=https://bitvpn.vercel.app?api=https://ВАШ_ДОМЕН

   И поднимите API мини-апп на сервере (см. docs/deploy/КАК_НАЙТИ_ЛОГИ_НА_СЕРВЕРЕ.txt).
   После этого оплата в Mini App будет идти через ваш сервер и логи
   будут в journalctl.

4. ПРОВЕРКА

   • С переменными на Vercel: пересоберите проект (Redeploy), снова
     откройте Mini App и нажмите способ оплаты.
   • С API на сервере: откройте Mini App (кнопка «Открыть VPN»),
     нажмите оплату — в journalctl -u miniapp-api -f должны появиться
     логи создания платежа или ошибка ЮKassa.
