# Проверка оплаты и выдачи ссылок (автоматизация)

Проведена проверка цепочки: **оплата (бот и мини-апп) → выдача ссылки Happ (или конфига)**. Ниже — что было, что сделано и что нужно настроить у себя.

---

## Как было (до правок)

| Где | Что происходило |
|-----|------------------|
| **Мини-апп** | Пользователь выбирает тариф и способ оплаты → вызывается `POST /api/miniapp/create-payment` → создаётся запись в БД и платёж в ЮKassa → пользователю открывается ссылка на оплату (СБП/карта). |
| **После оплаты** | ЮKassa не уведомляла наш сервер. Ссылка на VPN выдавалась **только** если пользователь заходил в **бота** и нажимал **«Проверить платеж»**. Кто оплатил только из мини-апп и не нажимал эту кнопку в боте — ссылку не получал. |
| **Бот** | При нажатии «Проверить платеж» бот опрашивал ЮKassa (`check_payment`), при успехе создавал подписку, вызывал Happ API, отправлял ссылку и файл .txt в чат. Всё работало только при ручной проверке. |

Итог: автоматической выдачи ссылки после оплаты из мини-апп не было.

---

## Что сделано

### 1. Вебхук ЮKassa в API

- Добавлен endpoint **`POST /api/webhook/yookassa`** в `api_miniapp.py`.
- ЮKassa при успешной оплате отправляет уведомление на этот URL. Сервер:
  - находит платёж в БД по `object.id` (id платежа в ЮKassa);
  - помечает платёж выполненным, создаёт подписку;
  - при настроенных HAPP_* создаёт ссылку Happ и отправляет её пользователю в Telegram (сообщение + файл .txt);
  - при отсутствии Happ отправляет конфиг WireGuard (файл + QR);
  - отправляет сообщение «Настроить VPN» с кнопками выбора устройства;
  - всегда возвращает **200 OK** (требование ЮKassa).

Теперь ссылка (или конфиг) приходит в бот **автоматически** после успешной оплаты, без нажатия «Проверить платеж».

### 2. Подсказка в мини-апп

- После открытия ссылки на оплату показывается сообщение:  
  **«После оплаты ссылка на VPN придёт вам в бот автоматически.»**  
  Обновлены: `public/index.html`, `webapp/index.html`, `api/root_index.html`, `index.html`.

## Что нужно сделать вам

### 1. Включить и настроить вебхук в ЮKassa

1. Зайдите в [личный кабинет ЮKassa](https://yookassa.ru/my).
2. Раздел **«Настройки» → «Уведомления»** (или «HTTP-уведомления»).
3. Укажите URL вебхука:  
   `https://ВАШ_ДОМЕН_API/api/webhook/yookassa`  
   Например: `https://nikolay.lisobyk.fvds.ru/api/webhook/yookassa` (если API крутится там).
4. Включите событие **«Платёж успешно проведён»** (payment.succeeded).
5. Сохраните. ЮKassa будет отправлять POST-запросы на этот URL при успешной оплате.

Важно: URL должен быть доступен по **HTTPS** снаружи. Если мини-апп и API на Vercel — нужен отдельный сервер с API (как у вас nikolay.lisobyk.fvds.ru), потому что на Vercel серверные функции имеют свой URL; вебхук нужно указать тот, где реально крутится `api_miniapp.py` с этим маршрутом.

### 2. Проверить переменные окружения на сервере API

На сервере, где запущен API (тот же, что принимает вебхук), в `.env` должны быть:

- **BOT_TOKEN** — чтобы отправлять сообщения в Telegram.
- **DATABASE_URL** — та же БД, что и у бота.
- **HAPP_API_URL**, **HAPP_PROVIDER_CODE**, **HAPP_AUTH_KEY**, **HAPP_SUBSCRIPTION_URL** — для выдачи ссылки Happ (если не заданы, будет выдан конфиг WireGuard).
- **YOOKASSA_SHOP_ID**, **YOOKASSA_SECRET_KEY** — уже используются для создания платежей.

### 3. Двойная выдача (вебхук и «Проверить платеж»)

- Если пользователь **сначала** оплатил, вебхук уже создал подписку и отправил ссылку.
- Если он потом в боте нажмёт **«Проверить платеж»**, бот увидит `status == 'completed'` и снова попытается создать подписку — в коде бота подписка создаётся только при переходе из `pending` в `completed`, и запись о платеже обновляется, поэтому повторная выдача не делается (подписка уже есть). В текущей реализации при повторном нажатии «Проверить платеж» бот просто проверяет статус; логика в `verify_payment` создаёт подписку только один раз при первом переходе в `completed`. Так что двойной выдачи ссылки не будет.

(При желании можно в боте при уже выполненном платеже показывать сообщение «Оплата уже проведена, ссылка была отправлена ранее» и не слать файлы повторно — это улучшение UX, не обязательно для автоматизации.)

## Краткая схема после настройки

1. Пользователь в **мини-апп** выбирает тариф и способ оплаты → создаётся платёж в ЮKassa, открывается форма оплаты.
2. Пользователь платит (СБП/карта).
3. ЮKassa отправляет уведомление на **`/api/webhook/yookassa`**.
4. API находит платёж, создаёт подписку, получает ссылку Happ (или конфиг), отправляет в Telegram пользователю сообщение + файл + «Настроить VPN».
5. Пользователь получает всё в боте без нажатия «Проверить платеж».

Параллельно кнопка **«Проверить платеж»** в боте по-прежнему работает (для тех, кто создавал платёж в боте или хочет проверить вручную).
