  СБП не отображается — что проверить

  В коде уже передаётся  capture: true  (одностадийная оплата), как
  просит ЮKassa. Если СБП не появился — значит на сервере или Vercel
  запущена старая версия без этого параметра. Обязательно задеплойте.

1. ЗАДЕПЛОЙТЕ НОВЫЙ КОД

   • Сервер (бот): запустите  Запустить_деплой_на_сервер.vbs
     или  ДЕПЛОЙ_НА_СЕРВЕР.bat  из папки проекта.
     После деплоя бот должен показывать две кнопки: «ЮKassa» и «СБП».

   • Vercel (Mini App API): если Mini App открывается с bitvpn.vercel.app,
     сделайте пуш в GitHub — Vercel подхватит изменения и в списке
     способов оплаты появится СБП.

2. ГДЕ ДОЛЖНЫ БЫТЬ КНОПКИ

   • В боте: Купить VPN → выбрать тариф (срок/устройства) →
     «Выберите способ оплаты» → должны быть «ЮKassa» и «СБП».

   • В Mini App: Оформить подписку → Оплатить X ₽ →
     «Выберите способ оплаты» → «ЮKassa» и «СБП».

3. СБП В ЛИЧНОМ КАБИНЕТЕ ЮKASSA

   Если кнопка «СБП» есть, но при нажатии ошибка или не открывается
   оплата по СБП:

   • Зайдите на  https://yookassa.ru/my
   • Настройки магазина → Способы оплаты (или «Подключенные методы»).
   • Убедитесь, что СБП (Система быстрых платежей) подключена.
   • Если СБП нет в списке — подключите в разделе приёма платежей
     или напишите в поддержку ЮKassa.

4. ПРОВЕРКА ПОСЛЕ ДЕПЛОЯ

   • В боте: /start → Купить VPN → выбрать 1 мес, 1 устройство →
     в сообщении «Выберите способ оплаты» должны быть две кнопки.
   • Нажатие «СБП» открывает страницу ЮKassa сразу с QR/списком банков СБП.

5. ГДЕ СМОТРЕТЬ ЛОГИ (если ошибка «Не удалось создать платёж»)

   Мини-приложение вызывает API на Vercel (тот же домен, что и страница).
   Запрос create-payment обрабатывается там — логи смотрите в Vercel.

   • Vercel (логи API Mini App):
     — Зайдите на  https://vercel.com  → ваш аккаунт → проект (VPN BOT).
     — Вкладка  Logs  (или  Deployments → выберите последний деплой →
       Runtime Logs / Function Logs).
     — Включите фильтр по времени, нажмите СБП в мини-приложении и
       посмотрите новые записи: там будет строка с ошибкой ЮKassa
       (например «YooKassa API error: 400 ...» или текст из ответа API).

   • Свой сервер (логи бота, если API мини-приложения крутится там):
     — Подключитесь по SSH:  ssh root@213.165.38.222
     — Логи сервиса:  journalctl -u vpn-bot -f --no-pager
     — Нажмите СБП в приложении и смотрите вывод в реальном времени.
     — Выйод: Ctrl+C.

   По логам будет видно точный ответ ЮKassa (код и описание ошибки).

6. ЕСЛИ ЛОГОВ НА VERCEL ВСЁ РАВНО НЕТ

   • Задеплойте последний код (в нём добавлен print при каждом вызове
     create-payment). Пуш в GitHub → подождите деплой → сбросьте фильтры
     (Reset Filters), период «Last hour», снова нажмите СБП. Ищите строку
     [MINIAPP] create-payment called или create-payment ERROR.
   • В Vercel зайдите в Observability (если есть) или в конкретный деплой
     → Runtime Logs — иногда логи видны только там.
   • Вариант «логи на своём сервере»: поднять API мини-апп на сервере
     и направлять запросы туда — тогда всё будет в journalctl. Подробно
     в docs/deploy/КАК_НАЙТИ_ЛОГИ_НА_СЕРВЕРЕ.txt (раздел miniapp-api).
