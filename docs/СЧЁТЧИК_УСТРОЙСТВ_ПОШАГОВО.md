# Счётчик «Подключено» (0/10) — пошагово

Если счётчик показывает 0/10 или **вообще не меняется** после подключения устройства в Happ, сделайте шаги ниже **по порядку**.

---

## Почему счётчик не отображается или не меняется

| Причина | Что сделать |
|--------|--------------|
| **Неверный HAPP_API_URL** | В `.env` на **сервере** должно быть `HAPP_API_URL=https://happ-proxy.com`. Для `list-install` (счётчик) работает именно этот адрес; `https://api.happ-proxy.com` даёт 404. После смены — перезапустить API. |
| **Домен редиректа не зарегистрирован в Happ** | Один раз выполнить `python scripts/happ_add_redirect_domain.py` (см. Шаг 0). Иначе Happ не обновляет счётчик при добавлении ссылки. |
| **В Happ добавлена не та ссылка** | В Happ нужно добавлять **ссылку из приложения** (Устройства → «Скопировать ссылку»), а не прямую ссылку на сервер подписки (95.181.175.67). |
| **Ссылка без кода (happ:// или старый формат)** | Ссылка должна быть вида `https://ваш-домен/sub/XXXXXXXXXXXX` (12 символов кода). Если в боте/приложении выдаётся старая ссылка — перевыпустите подписку или проверьте `HAPP_SUBSCRIPTION_REDIRECT_BASE` и перезапуск API. |

Проверка с сервера: откройте в браузере  
`https://ваш-API/api/miniapp/debug-install-stats?install_code=КОД_ИЗ_ССЫЛКИ`  
(12 символов после `/sub/`). Если **found: true** и **install_count** > 0 — API и Happ настроены верно; обновите экран «Устройства» (↻).

---

## Шаг 0. Один раз зарегистрировать домен редиректа в Happ (частая причина)

Когда пользователь добавляет ссылку в Happ, приложение отправляет в Happ **домен** этой ссылки (например `155.212.164.135`). Если этот домен не зарегистрирован у вас в Happ, счётчик **никогда не обновится**.

**Сделайте один раз** (на компьютере, где есть проект с настроенным `.env`):

```bash
cd "d:\VPN BOT"
python scripts/happ_add_redirect_domain.py
```

Должно вывести: `OK: домен ... зарегистрирован` или `уже зарегистрирован`. Если домен уже был добавлен — **всё равно важно**: в Happ на телефоне **удалите** эту подписку и **добавьте её заново** (ссылку из приложения, шаг 1). Иначе старое добавление могло не привязаться к счётчику.

---

## Шаг 1. Узнать правильную ссылку

Счётчик работает **только** для той ссылки, которую выдаёт приложение. Она должна быть вида:

- `https://nikolay.lisobyk.fvds.ru/sub/XXXXXXXXXXXX`  
  или  
- `https://155.212.164.135/sub/XXXXXXXXXXXX`

**Где взять:**

1. Откройте мини-приложение бота (из меню бота).
2. Зайдите в раздел **«Устройства»** (или «Подключено»).
3. Нажмите **«+ Добавить устройство»**.
4. Скопируйте ссылку из экрана (кнопка «Скопировать» или поле со ссылкой) — **это и есть та ссылка**, которую нужно добавлять в Happ.

Запомните или сохраните её. Дальше в Happ нужно добавлять **именно её**, а не ссылку вида `https://95.181.175.67:2096/...`.

---

## Шаг 2. Добавить эту ссылку в Happ

1. Установите приложение **Happ**, если ещё не установлено.
2. Откройте Happ.
3. Зайдите в **Подписки** (или «Subscriptions»).
4. Нажмите **«+»** (добавить подписку).
5. Вставьте **ту самую ссылку** из шага 1 (из мини-приложения, с `/sub/` и кодом).
6. Сохраните/подключитесь.

Подождите **1–2 минуты** — Happ иногда обновляет счётчик не сразу.

---

## Шаг 3. Обновить счётчик в приложении

1. Снова откройте мини-приложение бота.
2. Зайдите в **«Устройства»**.
3. Нажмите кнопку **«↻»** (обновить) рядом с «0/10».

Должно показаться 1/10 (или больше, если подключали несколько раз с той же ссылкой).

---

## Шаг 4. Если всё ещё 0/10 — проверить через браузер

Нужно убедиться, что Happ «видит» ваш код.

1. Из ссылки из шага 1 возьмите **12 символов после** `/sub/`.  
   Пример: ссылка `https://nikolay.lisobyk.fvds.ru/sub/yHmESPsZKd76` → код **yHmESPsZKd76**.
2. Откройте в браузере (подставьте **свой** код вместо `yHmESPsZKd76`):

   ```
   https://nikolay.lisobyk.fvds.ru/api/miniapp/debug-install-stats?install_code=yHmESPsZKd76
   ```

3. Смотрите ответ:
   - **found: true**, **install_count: 1** (или больше) — всё ок, после обновления (↻) в приложении счётчик должен показать правильное число.
   - **found: false** — в Happ нет такой записи. Значит, в Happ добавлена **другая** ссылка (не из шага 1). Вернитесь к шагу 1 и шагу 2 и добавьте в Happ **именно** ссылку из мини-приложения.
   - **message про HAPP_API_URL / HAPP_PROVIDER_CODE / HAPP_AUTH_KEY** — на сервере не настроены переменные Happ. Переходите к шагу 5.

---

## Шаг 5. Если в шаге 4 пишет, что не заданы HAPP_… (на сервере)

API работает на сервере **nikolay.lisobyk.fvds.ru**. На **этом же сервере** в папке, откуда запускается API, должен быть файл **.env** с такими строками (значения — ваши из кабинета Happ):

```env
HAPP_API_URL=https://api.happ-proxy.com
HAPP_PROVIDER_CODE=ваш_код
HAPP_AUTH_KEY=ваш_ключ
```

Что сделать:

1. Подключиться к серверу (SSH).
2. Перейти в папку проекта (где лежит `api_miniapp.py` и т.п.).
3. Открыть или создать файл `.env` и проверить/добавить эти три строки.
4. Сохранить файл.
5. Перезапустить API, например:
   ```bash
   sudo systemctl restart miniapp-api
   ```
   (Имя сервиса может быть другим — как вы его настроили.)

После этого снова выполните шаг 4 (проверка в браузере) и шаг 3 (↻ в приложении).

---

## Кратко

| Что делать | Зачем |
|------------|--------|
| **Один раз** запустить `python scripts/happ_add_redirect_domain.py` | Зарегистрировать домен редиректа в Happ — иначе счётчик не обновляется |
| Взять ссылку из мини-приложения («Устройства» → «Добавить устройство» → скопировать) | Это единственная ссылка, по которой считается счётчик |
| Добавить **эту** ссылку в Happ (Подписки → +) | Иначе Happ считает другую «установку», и счётчик 0 |
| Подождать 1–2 минуты и нажать ↻ в «Устройства» | Обновить данные с Happ |
| Открыть debug-ссылку в браузере со своим кодом | Убедиться, что Happ видит ваш код (found: true) |
| На сервере проверить .env и перезапустить API | Чтобы API мог запросить счётчик у Happ |

**Проверка list-install с вашего ПК:**  
`python scripts/test_happ_list_install.py` — выведет все записи из Happ.  
`python scripts/test_happ_list_install.py ВАШ_КОД_12_СИМВОЛОВ` — проверит, есть ли ваш код в списке.

Если после всех шагов в debug **found: true** и **install_count: 1**, а в приложении всё ещё 0 — напишите, на каком шаге вы сейчас и что именно видите (в приложении и в ответе debug-ссылки).
