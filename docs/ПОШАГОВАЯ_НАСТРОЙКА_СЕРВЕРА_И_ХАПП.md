# Пошаговая настройка: сервер, Happ, бот и мини-апп

Кратко: у вас **два сервера**, один — подписка (VPN), второй — бот и API. Ниже по шагам: что где нажать и что в какой файл поставить.

---

## Схема: кто за что отвечает

| Что | Где | Роль |
|-----|-----|------|
| **95.181.175.67** | Ваш купленный VPS | **Сервер подписки (VPN).** На нём стоит 3x-ui (или другая панель). Отдаёт подписку по ссылке вида `https://95.181.175.67:2096/sub_bitvpn/XXXXX`. Эту подписку вы **выдаёте людям** (через бота). |
| **155.212.164.135** | Сервер с ботом | **Бот + API мини-аппа.** Домен `nikolay.lisobyk.fvds.ru` указывает сюда. Здесь крутится Telegram-бот и API для мини-аппа. Бот вызывает Happ API и выдаёт пользователям **разные ссылки под тариф** (1 устройство, 3 и т.д.). |

Итого: подписка «живёт» на **95.181.175.67**, а бот и логика выдачи ссылок — на **155.212.164.135**.

## Часть 1. Neon (база данных)

Бот хранит пользователей, подписки и платежи в PostgreSQL. Удобно использовать Neon (бесплатный тир).

### Шаги в Neon

1. Зайдите на [neon.tech](https://neon.tech), войдите или зарегистрируйтесь.
2. **Create a project** (или «Новый проект»):
   - Имя — любое (например `vpn-bot`).
   - Регион — любой (например US East).
   - Нажмите **Create project**.
3. На странице проекта откройте вкладку **Connection** (или «Подключение»).
4. Выберите способ **Connection string** (не Pooler, если не нужен — можно и Pooler).
5. Скопируйте строку подключения. Она выглядит так:
   ```text
   postgresql://USER:PASSWORD@HOST/dbname?sslmode=require
   ```
   Иногда в конце ещё `&channel_binding=require` — оставьте как есть.

### Куда вставить в проекте

- В файле **`.env`** (в корне проекта, рядом с ботом) найдите строку:
  ```env
  DATABASE_URL=...
  ```
  Замените её на скопированную строку из Neon (без кавычек):
  ```env
  DATABASE_URL=postgresql://neondb_owner:ВАШ_ПАРОЛЬ@ep-xxxxx-pooler.us-east-1.aws.neon.tech/neondb?sslmode=require
  ```
- На **сервере**, где крутится бот (155.212.164.135), в файле `/opt/vpn-bot/.env` должна быть **та же** строка `DATABASE_URL`.

Таблицы бот создаёт сам при первом запуске (если в коде есть `create_tables()`). Отдельно в Neon ничего нажимать не нужно — только дать строку подключения.

## Часть 2. Happ (лимитированные ссылки под тариф)

Happ — это приложение, куда пользователь вставляет ссылку подписки. Через API Happ вы создаёте для каждого пользователя **свою ссылку с лимитом устройств** (1, 3, 5 и т.д. — по тарифу).

### Шаги в кабинете Happ

1. Зайдите на [happ-proxy.com](https://happ-proxy.com) или [happ.su](https://www.happ.su).
2. Войдите как **провайдер** (не как обычный пользователь приложения).
3. В личном кабинете найдите раздел **API** / **Для разработчиков** / **Ключи**:
   - **provider_code** — 8 символов (латиница/цифры).
   - **auth_key** — 32 символа.
   Скопируйте оба значения — они понадобятся в `.env`.
4. **Подписка (откуда раздавать):**
   - Если подписку отдаёт **ваш сервер 95.181.175.67** (3x-ui):
     - В 3x-ui создайте inbound и получите **ссылку подписки** (обычно в настройках inbound или подписок).
     - Она выглядит как: `https://95.181.175.67:2096/sub_bitvpn/XXXXXXXX` (порт и путь могут отличаться).
     - Эту ссылку **без** `?installid=...` вы поставите в `HAPP_SUBSCRIPTION_URL` в `.env`.
   - Если в кабинете Happ есть поле «URL подписки» / «Введите URL» — туда можно ввести **тот же** базовый URL: `https://95.181.175.67:2096/sub_bitvpn/XXXXXXXX` (без параметров). Бот сам будет добавлять `installid` через API.
5. **Домен подписки (если попросят):**  
   По документации Happ иногда нужно «привязать» домен подписки к вашему provider. Если подписка по IP (`95.181.175.67`), уточните в поддержке Happ, как привязать IP или нужен ли add-domain. Обычно для работы API достаточно корректных `provider_code` и `auth_key` и правильного базового URL подписки.

### Что поставить в .env (Happ)

В **`.env`** в корне проекта (и тот же блок — на сервере в `/opt/vpn-bot/.env`):

```env
HAPP_API_URL=https://api.happ-proxy.com
HAPP_PROVIDER_CODE=ваши_8_символов
HAPP_AUTH_KEY=ваши_32_символа
HAPP_SUBSCRIPTION_URL=https://95.181.175.67:2096/sub_bitvpn/ВАШ_ИДЕНТИФИКАТОР
```

- **HAPP_API_URL** — всегда `https://api.happ-proxy.com` (именно с `api.`, без него API даёт 404).
- **HAPP_PROVIDER_CODE** и **HAPP_AUTH_KEY** — из кабинета Happ (шаг 3 выше).
- **HAPP_SUBSCRIPTION_URL** — **базовая** ссылка подписки с **95.181.175.67** (из 3x-ui), **без** `?installid=...`. Бот сам добавит `?installid=КОД` при выдаче.

Чтобы пользователь не видел прямой URL подписки (только короткую ссылку с вашего API), добавьте:

```env
HAPP_SUBSCRIPTION_REDIRECT_BASE=https://155.212.164.135
```

Тогда пользователь будет получать ссылку вида `https://155.212.164.135/sub/XXXXXXXXXXXX` (редирект на ваш 95.181.175.67 с подставленным `installid`). Если не задать — бот будет выдавать прямую ссылку `https://95.181.175.67:2096/...?installid=...`.

**Важно:** если вы поставите в `HAPP_SUBSCRIPTION_URL` ссылку вида **`happ://crypt5/...`** (прямая ссылка из кабинета Happ), то бот будет выдавать **одну и ту же** ссылку всем (без разницы по тарифу). Разные ссылки под тариф получаются только при использовании **HTTP/HTTPS** базового URL (как выше) и вызове API Happ `add-install`.

## Часть 3. Файлы проекта: что куда поставить

### 3.1 Файл `.env` (в корне проекта и на сервере)

Соберите один блок (подставьте свои значения):

```env
# Telegram
BOT_TOKEN=токен_от_BotFather
ADMIN_IDS=ваш_telegram_id

# База — строка из Neon (часть 1)
DATABASE_URL=postgresql://user:password@host/db?sslmode=require

# Мини-апп: страница на Vercel, запросы на ваш API
WEBAPP_URL=https://bitvpn.vercel.app?api=https://nikolay.lisobyk.fvds.ru
MINIAPP_API_URL=https://nikolay.lisobyk.fvds.ru
BOT_USERNAME=Bitvpnproxy_bot

# Платежи (ЮKassa)
YOOKASSA_SHOP_ID=...
YOOKASSA_SECRET_KEY=...

# Happ (часть 2)
HAPP_API_URL=https://api.happ-proxy.com
HAPP_PROVIDER_CODE=ваш_8_символов
HAPP_AUTH_KEY=ваш_32_символа
HAPP_SUBSCRIPTION_URL=https://95.181.175.67:2096/sub_bitvpn/ВАШ_ПУТЬ
HAPP_SUBSCRIPTION_REDIRECT_BASE=https://155.212.164.135

# Тарифы (руб.)
PLAN_1_MONTH_PRICE=100
PLAN_3_MONTH_PRICE=285
PLAN_6_MONTH_PRICE=540
PLAN_9_MONTH_PRICE=765
PLAN_12_MONTH_PRICE=960
```

- `MINIAPP_API_URL` и домен в `WEBAPP_URL?api=...` — тот сервер, где крутится API (у вас **155.212.164.135** / nikolay.lisobyk.fvds.ru).
- `HAPP_SUBSCRIPTION_REDIRECT_BASE` — тот же сервер (155.212.164.135), чтобы выдавать ссылки вида `https://155.212.164.135/sub/КОД`.

### 3.2 Файл `deploy_config.cmd` (на вашем ПК, для деплоя)

Отвечает за то, **куда** пушить код и **откуда** копировать `.env`:

```cmd
set SERVER_USER=root
set SERVER_IP=155.212.164.135
set BOT_PATH=/opt/vpn-bot
set RESTART_CMD=sudo systemctl restart vpn-bot
set GIT_BRANCH=main
set COPY_ENV_TO_SERVER=1
```

- **SERVER_IP** — сервер, где крутится **бот и API** (155.212.164.135), **не** 95.181.175.67.
- **COPY_ENV_TO_SERVER=1** — при деплое ваш локальный `.env` копируется на сервер (в нём должны быть актуальные Neon, Happ и т.д.).

### 3.3 На сервере 95.181.175.67 (VPN / подписка)

- Установите 3x-ui (или вашу панель).
- Создайте inbound, получите **ссылку подписки**.
- Эту ссылку (без `installid`) вы уже прописали в `HAPP_SUBSCRIPTION_URL` в `.env` (часть 2 и 3.1).

На **95.181.175.67** бот и код проекта **не ставятся** — там только панель подписки.

### 3.4 На сервере 155.212.164.135 (бот + API)

- Установлены: бот (systemd `vpn-bot`), API мини-аппа (systemd `miniapp-api`), nginx, при необходимости certbot.
- В `/opt/vpn-bot/.env` — тот же `.env`, что и у вас в проекте (при `COPY_ENV_TO_SERVER=1` он обновляется при деплое).
- Nginx настроен так, чтобы запросы к домену (nikolay.lisobyk.fvds.ru) на `/api/` и `/sub/` шли на API (порт 8765).

## Краткий чеклист

| Шаг | Где | Действие |
|-----|-----|----------|
| 1 | Neon | Создать проект → Connection → скопировать строку подключения. |
| 2 | `.env` | Вписать `DATABASE_URL` из Neon. |
| 3 | Happ | Войти как провайдер → взять `provider_code` и `auth_key`. |
| 4 | 3x-ui на 95.181.175.67 | Получить ссылку подписки (без параметров). |
| 5 | `.env` | Вписать `HAPP_API_URL`, `HAPP_PROVIDER_CODE`, `HAPP_AUTH_KEY`, `HAPP_SUBSCRIPTION_URL` (ссылка с 95.181.175.67), при желании `HAPP_SUBSCRIPTION_REDIRECT_BASE=https://155.212.164.135`. |
| 6 | `deploy_config.cmd` | Указать `SERVER_IP=155.212.164.135`, `BOT_PATH`, `COPY_ENV_TO_SERVER=1`. |
| 7 | Сервер 155.212.164.135 | Убедиться, что бот и API запущены, в `/opt/vpn-bot/.env` актуальный `.env` (деплой с копированием .env или вручную). |

После этого при оплате бот будет вызывать Happ API, создавать лимитированную ссылку под тариф (1/3/5/10 устройств) и выдавать пользователю ссылку (либо `https://155.212.164.135/sub/КОД`, либо прямую на 95.181.175.67 с `installid` — в зависимости от `HAPP_SUBSCRIPTION_REDIRECT_BASE`).
