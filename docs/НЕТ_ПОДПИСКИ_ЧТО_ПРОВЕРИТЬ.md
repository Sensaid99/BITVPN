# В мини-аппе пишет «Нет подписки» или «Ошибка загрузки» — что проверить

Мини-апп показывает **«Нет подписки»**, когда запрос к API `/api/miniapp/me` вернул данные, но без активной подписки. **«Ошибка загрузки»** появляется, когда запрос к API не удался (сеть, 401, 500) или приложение открыто не из Telegram. Ниже — по шагам, что проверить.

---

## Если видите «Ошибка загрузки»

Значит запрос к **API** (ваш сервер) по адресу `/api/miniapp/me` не вернул успешный ответ. Проверьте по порядку:

1. **Открываете приложение из бота в Telegram**  
   Не по прямой ссылке в браузере. Зайдите в @Bitvpnproxy_bot и откройте приложение кнопкой меню (или ссылкой из бота). Иначе не будет initData и API вернёт ошибку.

2. **В ссылке приложения есть параметр `?api=...`**  
   Кнопка меню бота должна открывать ссылку вида:  
   `https://bitvpn.vercel.app?api=https://bitecosystem.ru`  
   В BotFather в настройках кнопки меню (Web App) должен быть указан **полный** URL с `?api=https://ваш-домен-api`. То же значение задаётся в `.env` в **WEBAPP_URL**. Если `api=` нет, приложение будет слать запросы на Vercel, а не на ваш API — будет «Ошибка загрузки».

3. **API доступен по этому адресу**  
   Откройте в браузере (с телефона или ПК):  
   `https://bitecosystem.ru/api/miniapp/plans`  
   Должен открыться JSON с планами. Если страница не открывается или ошибка — API не доступен (сервер, nginx, домен).

4. **На сервере правильный BOT_TOKEN**  
   В `.env` на сервере (где крутится miniapp-api) должен быть тот же **BOT_TOKEN**, что у бота в BotFather. Если токен другой, API вернёт 401 на `/api/miniapp/me` и приложение покажет «Ошибка загрузки». После смены `.env` перезапустите: `sudo systemctl restart miniapp-api`.

5. **Повторить попытку**  
   В приложении нажмите на надпись «Ошибка загрузки» — должен отправиться повторный запрос. Если после проверки п. 1–4 снова ошибка — посмотрите логи API на сервере: `sudo journalctl -u miniapp-api -n 50 --no-pager`.

---

## 1. Открываете приложение из Telegram

Приложение должно открываться **из чата с ботом** (кнопка меню или ссылка в боте). Тогда Telegram передаёт в запрос данные авторизации (initData). Если открыть мини-апп по прямой ссылке в обычном браузере, initData не будет — API вернёт ошибку, и может показываться «Нет подписки» или «Ошибка загрузки».

**Что сделать:** зайти в @Bitvpnproxy_bot (ваш бот) и открыть приложение оттуда.

---

## 2. Один и тот же BOT_TOKEN на боте и на API

API проверяет initData по **BOT_TOKEN**. Токен в `.env` на сервере (где крутится API) должен **совпадать** с токеном того бота, из которого вы открываете мини-апп.

**Что сделать:** в `deploy_config.cmd` включено копирование `.env` на сервер (`COPY_ENV_TO_SERVER=1`). Убедитесь, что в **локальном** `.env` в корне проекта указан правильный `BOT_TOKEN` от @BotFather для этого бота, затем заново выполните деплой (чтобы этот `.env` попал на сервер).

---

## 3. В базе Neon есть вы и активная подписка

«Нет подписки» часто значит: пользователь есть, но у него **нет ни одной активной подписки** (все истекли или помечены неактивными).

**Что сделать:**

1. Узнайте свой **Telegram ID** (например, напишите боту @userinfobot в Telegram — он покажет ID).
2. Зайдите в **Neon** (neon.tech) → ваш проект → **SQL Editor**.
3. Выполните (подставьте свой `TELEGRAM_ID`):

   ```sql
   SELECT id, telegram_id, username FROM users WHERE telegram_id = TELEGRAM_ID;
   ```

   Если записей нет — вы ещё не «зарегистрированы» в боте. Напишите боту в Telegram **/start**, подождите пару секунд и обновите приложение.

4. Если пользователь есть, посмотрите подписки (подставьте `USER_ID` из предыдущего запроса):

   ```sql
   SELECT id, user_id, plan_type, is_active, end_date
   FROM subscriptions
   WHERE user_id = USER_ID
   ORDER BY end_date DESC;
   ```

   Чтобы подписка считалась активной, нужна строка с `is_active = true` и `end_date` **больше** текущего времени (по UTC). Если таких нет — в приложении будет «Нет подписки»; нужно оформить подписку заново или, если вы админ, проверить выдачу админской подписки.

## 4. Отладочный запрос API (почему «Нет подписки»)

На сервере добавлен эндпоинт **POST /api/miniapp/debug-subscription**: он принимает initData и отвечает коротким статусом — есть ли пользователь в базе и есть ли у него активная подписка.

**Как вызвать:**

1. Откройте мини-апп **из Telegram** (как в п. 1).
2. Если приложение открылось в браузере или в WebView с доступом к консоли — откройте консоль разработчика (F12 → вкладка Console).
3. В консоли выполните (одной строкой, замените `ВАШ_ДОМЕН_API` на домен вашего API, например `bitecosystem.ru`):

   ```javascript
   fetch('https://ВАШ_ДОМЕН_API/api/miniapp/debug-subscription', {
     method: 'POST',
     headers: { 'Content-Type': 'application/json' },
     body: JSON.stringify({ initData: Telegram.WebApp.initData })
   }).then(r => r.json()).then(console.log);
   ```

   Если переменной `Telegram` нет (например, открыли не из Telegram), этот способ не сработает — тогда проверяйте базу (п. 3) и BOT_TOKEN (п. 2).

В ответе будет одно из:

- **initData_required** / **invalid_initData** — проблема с открытием из Telegram или с BOT_TOKEN на API.
- **user_not_in_db** — в базе нет пользователя с вашим Telegram ID; сделайте /start в боте и обновите приложение.
- **no_active_subscription** — пользователь есть, активной подписки нет; проверьте таблицу `subscriptions` в Neon (п. 3).
- **has_active_subscription** — подписка в базе есть; если в приложении всё равно «Нет подписки», обновите страницу или нажмите «Повторить» в приложении.

## 5. Краткий чеклист

| Проверка | Действие |
|----------|----------|
| Открытие из бота | Открывать мини-апп только из чата с вашим ботом в Telegram. |
| BOT_TOKEN | В `.env` на сервере тот же BOT_TOKEN, что у бота; после смены — деплой и перезапуск API. |
| Пользователь в базе | В Neon в таблице `users` есть строка с вашим `telegram_id`; если нет — написать боту /start. |
| Активная подписка | В таблице `subscriptions` есть запись с `user_id` этого пользователя, `is_active = true` и `end_date` в будущем. |
| Отладка | Вызвать POST `/api/miniapp/debug-subscription` с initData и посмотреть причину в ответе. |

После проверки п. 1–3 и при необходимости п. 4 обычно удаётся понять, из‑за чего показывается «Нет подписки», и исправить (настроить токен, обновить базу или оформить подписку).
