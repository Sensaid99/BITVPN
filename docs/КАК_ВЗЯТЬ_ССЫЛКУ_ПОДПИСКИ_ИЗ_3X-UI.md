# Как взять ссылку подписки из 3x-ui (пошагово)

Сервер подписки у вас — **95.181.175.67**. На нём установлена панель **3x-ui**. Из неё нужно скопировать **одну ссылку** (базовую, без `?installid=...`) и вставить её в файл `.env` в переменную `HAPP_SUBSCRIPTION_URL`.

Ниже — куда заходить и что нажимать, по шагам.

---

## Шаг 1. Открыть панель 3x-ui в браузере

1. Откройте браузер (Chrome, Firefox, Edge — любой).
2. В адресной строке введите:
   ```text
   https://95.181.175.67:2096
   ```
   Или если панель у вас открывается на другом порту, то:
   ```text
   https://95.181.175.67:2053
   ```
   (часто бывает **2053** — панель управления, или **2096** — порт подписки; попробуйте оба, если один не откроется).
3. Нажмите **Enter**.
4. Если браузер пишет «Небезопасное соединение» или «Ваше подключение не защищено» — нажмите **«Дополнительно»** (или «Advanced») и выберите **«Перейти на сайт»** (или «Proceed to …»). Это нормально для доступа по IP без своего сертификата.
5. Должна открыться страница входа 3x-ui (логин и пароль).

---

## Если панель не открывается (оба адреса не работают)

Сделайте по порядку:

### 1. Попробовать **http** вместо https

В браузере введите **без** буквы «s»:
- `http://95.181.175.67:2053`
- `http://95.181.175.67:2096`

Иногда панель настроена только по HTTP.

### 2. Проверить, что сервер вообще отвечает

- Откройте сайт [https://ping.eu/port-check/](https://ping.eu/port-check/) (или любой «проверка порта»).
- Введите IP: **95.181.175.67**, порт: **2053**, нажмите проверку.
- То же самое для порта **2096**.

Если оба порта «закрыты» или «не отвечают» — до сервера не достучаться (фаервол у хостера или у вас, или панель не запущена).

### 3. Есть ли у вас доступ по SSH к серверу 95.181.175.67?

Если **да** (знаете логин/пароль или ключ):

1. Подключитесь (в Windows: PuTTY или «Терминал» в VS Code; в поле host введите `95.181.175.67`, порт 22).
2. Войдите под пользователем (часто `root`).
3. Выполните команды и пришлите результат (или посмотрите сами):

   **Узнать, какие порты слушает панель:**
   ```bash
   ss -tlnp | grep -E '2053|2096|x-ui'
   ```
   или:
   ```bash
   netstat -tlnp | grep -E '2053|2096'
   ```

   **Проверить, установлена ли 3x-ui и запущена ли:**
   ```bash
   systemctl status x-ui
   ```
   или:
   ```bash
   systemctl status 3x-ui
   ```

   Если сервис есть и «active» — панель запущена. Номер порта смотрите в настройках (часто в файле конфига или в выводе `ss`/`netstat`).

4. **Фаервол на сервере:** откройте порты (подставьте свой порт, например 2053):
   ```bash
   ufw allow 2053/tcp
   ufw allow 2096/tcp
   ufw reload
   ```
   Или если используете iptables — нужно добавить правило для этих портов.

Если **SSH нет** — тогда доступ к 95.181.175.67 должен настроить тот, кто устанавливал сервер (или хостинг). Без доступа к серверу или к тому, кто им управляет, открыть панель по новому адресу не получится.

### 3.1 Если 2053 закрыт, а 2096 открыт и по 2096 в браузере «404»

Такое бывает: порт **2053** (панель входа) закрыт или панель не запущена, а порт **2096** (сервис подписок) открыт. По адресу `https://95.181.175.67:2096` сервер отдаёт «404 page not found» — это нормально: подписка отдаётся не с корня, а по пути вроде `/sub_bitvpn/...`.

**Что сделать:** откройте в браузере **полную** ссылку подписки (с путём и идентификатором), а не просто `:2096`:
```text
https://95.181.175.67:2096/sub_bitvpn/bgmdn1s016p08yfb
```
- Если откроется страница или начнётся загрузка — ссылка живая. Пропишите её в `.env` в `HAPP_SUBSCRIPTION_URL` (см. пункт 5 ниже) — панель для этого не нужна.
- Если по этой ссылке тоже 404 — путь или ID на сервере сменили; без доступа к панели или к тому, кто управляет сервером, новую ссылку не взять (нужно открыть 2053 и зайти в панель или попросить актуальную ссылку).

### 4. Панель может быть по домену, а не по IP

Если при установке 3x-ui вам давали ссылку вида `https://какой-то-домен.ru`, откройте именно её в браузере. Тогда базовая ссылка подписки может быть вида `https://этот-же-домен.ru:2096/sub_bitvpn/...` — её тоже можно использовать в `HAPP_SUBSCRIPTION_URL`.

### 5. Ссылка уже есть в проекте

Раньше в проекте использовалась ссылка:
`https://95.181.175.67:2096/sub_bitvpn/bgmdn1s016p08yfb`

Если вы **ничего не меняли** на сервере 95.181.175.67 и подписка там по-прежнему работает (пользователи подключаются), можно снова прописать в `.env`:
```env
HAPP_SUBSCRIPTION_URL=https://95.181.175.67:2096/sub_bitvpn/bgmdn1s016p08yfb
```
Проверка: откройте в браузере эту ссылку — должна открыться страница или начаться загрузка. Если открывается — эту ссылку можно использовать, заходить в панель только чтобы взять ссылку не обязательно.

## Шаг 2. Войти в панель

1. Введите **логин** (username) и **пароль**, которые вы задавали при установке 3x-ui.
2. Нажмите кнопку входа (Login / Войти).
3. После входа вы попадёте в главное меню панели (обычно слева или сверху есть пункты: Dashboard, Inbounds, Panel Settings и т.п.).

---

## Шаг 3. Найти настройки подписки (Subscription)

Нужно найти раздел, где задаётся **порт** и **путь** подписки. В 3x-ui это обычно в настройках панели.

1. В меню слева (или сверху) найдите пункт **«Panel Settings»** или **«Настройки панели»** / **«Settings»** и нажмите на него.
2. В открывшейся странице найдите блок с названием **«Subscription»** / **«Подписка»** / **«Subscription Server»** и откройте его (клик по заголовку или вкладке).
3. Там вы увидите поля, например:
   - **Subscription Port** (порт подписки) — часто **2096**;
   - **Subscription Path** или **URI Path** (путь) — например `/sub/` или `/sub_bitvpn/`;
   - может быть **Sub Domain** (домен) — если пусто, в ссылке будет IP.

Запомните или запишите:
- порт (например **2096**);
- путь (например **/sub_bitvpn/** или **/sub/**).

Ссылка подписки будет такой:
```text
https://95.181.175.67:ПОРТ/ПУТЬ
```
Путь пишем **без** слеша в конце, например: `/sub_bitvpn` или `sub_bitvpn`. Если в настройках указано `/sub_bitvpn/`, то в ссылке можно писать `https://95.181.175.67:2096/sub_bitvpn`.

Но нам нужна не просто «корневая» ссылка, а **ссылка на конкретную подписку** (на ваш inbound). Её обычно показывают в разделе **Inbounds**.

---

## Шаг 4. Взять ссылку подписки из Inbounds

1. В меню слева (или сверху) нажмите **«Inbounds»** (входящие подключения).
2. Откроется список **inbound’ов**. Выберите тот, который вы используете для пользователей (для VPN), и нажмите на него (или на иконку редактирования / карандаша).
3. В карточке этого inbound’а найдите:
   - либо кнопку/поле **«Subscription link»** / **«Ссылка подписки»**;
   - либо вкладку **«Subscription»**;
   - либо список клиентов (Clients), а у каждого клиента — ссылку.
4. Скопируйте **ссылку подписки**. Она может выглядеть так:
   ```text
   https://95.181.175.67:2096/sub_bitvpn/bgmdn1s016p08yfb
   ```
   или:
   ```text
   https://95.181.175.67:2096/sub/xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx
   ```
   Важно: копируйте ссылку **без** `?installid=...` и **без** чего-либо после `?`. То есть только:
   ```text
   https://95.181.175.67:2096/sub_bitvpn/bgmdn1s016p08yfb
   ```
   Это и есть **базовая ссылка подписки**.

Если в панели показывают ссылку уже с `?installid=...` — уберите всё, начиная с `?`, и оставьте только часть до знака вопроса.

---

## Шаг 5. Если ссылку нигде не видно — собрать её вручную

Иногда в интерфейсе показывают только порт и путь, а «длинную» ссылку не показывают. Тогда можно собрать её так:

1. Из **Panel Settings → Subscription** вы узнали:
   - порт (например **2096**);
   - путь (например **/sub_bitvpn/**).
2. В разделе **Inbounds** откройте нужный inbound и посмотрите:
   - есть ли у него **Subscription ID** или **Sub ID** (короткий код типа `bgmdn1s016p08yfb` или число);
   - или **Remark** / название, из которого иногда делают путь.
3. Соберите ссылку в формате:
   ```text
   https://95.181.175.67:2096/sub_bitvpn/ВАШ_ID
   ```
   Вместо **2096** подставьте свой порт подписки, вместо **sub_bitvpn** — свой путь (без лишних слэшей), вместо **ВАШ_ID** — тот идентификатор, который панель даёт этому inbound’у для подписки.

Проверка: откройте эту ссылку в браузере. Должна либо открыться страница с конфигом/подпиской, либо начаться загрузка файла. Если «страница не найдена» — значит путь или ID неверные, перепроверьте в Panel Settings и Inbounds.

---

## Шаг 6. Вставить ссылку в .env

1. На своём компьютере откройте папку с ботом (проект VPN BOT).
2. Откройте файл **`.env`** (блокнотом или редактором кода).
3. Найдите строку:
   ```env
   HAPP_SUBSCRIPTION_URL=...
   ```
4. Поставьте туда **скопированную базовую ссылку** (без `?installid=...`):
   ```env
   HAPP_SUBSCRIPTION_URL=https://95.181.175.67:2096/sub_bitvpn/bgmdn1s016p08yfb
   ```
   (подставьте свою ссылку, если она другая.)
5. Сохраните файл (Ctrl+S).

Готово. Эта ссылка — «база»: бот будет к ней добавлять параметр `?installid=КОД` для каждого пользователя, чтобы у каждого была своя лимитированная ссылка под тариф.

## Краткий чеклист

| № | Что сделать |
|---|-------------|
| 1 | Открыть в браузере `https://95.181.175.67:2096` (или :2053). |
| 2 | Войти по логину и паролю 3x-ui. |
| 3 | Зайти в **Panel Settings** → блок **Subscription**, запомнить порт и путь. |
| 4 | Зайти в **Inbounds** → открыть нужный inbound → найти и скопировать **ссылку подписки** (без `?installid=...`). |
| 5 | Если ссылки нет — собрать вручную: `https://95.181.175.67:ПОРТ/ПУТЬ/ID`. |
| 6 | В файле `.env` вписать эту ссылку в `HAPP_SUBSCRIPTION_URL=...` и сохранить. |

Если на каком-то шаге интерфейс у вас выглядит иначе (другая версия 3x-ui или язык) — напишите, что именно видите на экране (какие пункты меню), и можно будет подсказать точнее под ваш вариант.
