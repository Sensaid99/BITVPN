  ГДЕ ВКЛЮЧИТЬ COLD START И ДРУГИЕ НАСТРОЙКИ VERCEL
═══════════════════════════════════════════════════════════════════

■ Открыть настройки проекта
  1. Зайдите на https://vercel.com и войдите в аккаунт.
  2. Выберите проект (тот, где развёрнут мини-апп Bit VPN).
  3. Вверху откройте вкладку **Settings**.

■ Cold Start Prevention (чтобы мини-апп реже «засыпал»)
  1. В левом меню Settings нажмите **Functions** (или **Runtime**).
  2. Найдите блок **Runtime Settings** (или **Function Settings**).
  3. Найдите пункт **Cold Start Prevention**.
  4. Включите переключатель (On / Enabled).

■ Остальное (по желанию)
  • **Function Region** — уже стоит (например iad1). Менять не обязательно.
  • **Function CPU / Memory** — если мини-апп тормозит, можно поднять до 2 vCPU / 4 GB в том же блоке Runtime Settings.
  • **Node.js Version** — для вашего API (Python) не важно; можно не трогать.

■ Если не находите Cold Start Prevention
  В новых интерфейсах Vercel пункт может называться иначе или лежать в:
  Settings → **Deployments** → **Deployment Settings** (кнопка справа у последнего деплоя),
  или: Settings → **Functions** → внизу страницы **Runtime** / **Edge**.
  Включите всё, что связано с «Cold Start» или «Keep functions warm».
