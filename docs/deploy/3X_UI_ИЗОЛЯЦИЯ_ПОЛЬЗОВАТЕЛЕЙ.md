# 3x-ui / Xray: отдельный клиент и трафик на каждого пользователя

Бот и Happ выдают **одну ссылку подписки** с параметром `installid` (код установки Happ). На стороне **панели** трафик и лимиты считаются по **клиенту inbound** (email / UUID). Если в панели один общий клиент на всех — у всех будет общий счётчик (например «32 ГБ»).

## Что сделать, чтобы у каждого нового пользователя было «своё»

1. **Не используйте один и тот же email/клиент в inbound для всех.**  
   Для каждого реального пользователя в 3x-ui должен быть **отдельный клиент** в нужном inbound (или отдельная подписка, если у вас так заведено).

2. **Связка с Happ (`installid`).**  
   Ссылка вида `...?installid=КОД` должна в вашей схеме **привязывать** нового пользователя к **новому клиенту** в панели, а не к уже существующему с накопленным трафиком. Как именно это делает 3x-ui с `installid`, зависит от версии панели и типа inbound — проверьте в документации 3x-ui / тестом: новый `installid` → в панели появляется новый клиент или новая строка учёта.

3. **Сброс / отдельный учёт трафика.**  
   В настройках **клиента** (не только inbound): лимит трафика, сброс статистики — смотрите, чтобы новый клиент начинал **с нуля**. Старые тестовые клиенты с тем же email лучше удалить или не использовать для продакшена.

4. **Один inbound — много клиентов.**  
   Нормальная схема: один VLESS inbound, **много клиентов** (по одному на пользователя), в подписке отдаются **несколько нод** при необходимости (см. `ОДНА_ПОДПИСКА_НЕСКОЛЬКО_СЕРВЕРОВ_3XUI.md`).

5. **Проверка.**  
   После оплаты у нового пользователя откройте в панели список клиентов этого inbound и убедитесь, что для него **своя** строка и **нулевой** (или ожидаемый) расход, а не чужой.

## Если трафик «тянется» со старого пользователя

- Удалите/не переиспользуйте тестовые клиенты с тем же идентификатором.  
- Убедитесь, что бот не сохраняет в БД **одну и ту же** персональную ссылку двум пользователям (редкий сбой логики; проверка — разные `telegram_id` → разные записи подписки и ссылки).

## Мини-апп и бот

Код выдаёт **уникальный** `install_code` от Happ API (`add-install`) на пользователя и сохраняет ссылку в `subscriptions.vpn_config`. Если в панели при этом всё равно один клиент — проблема в **настройке 3x-ui / привязке installid**, а не в Python-боте.
