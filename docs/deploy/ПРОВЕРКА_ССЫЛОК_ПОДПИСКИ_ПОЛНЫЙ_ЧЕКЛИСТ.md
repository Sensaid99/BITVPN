# Полный чеклист: ссылки подписки — что где проверить и какие команды

Цель: убедиться, что **нигде не «висит» устаревшая логика**, публичная ссылка пользователя согласована с **панелью 3x-ui** и **API на bitecosystem.ru**, а мини-апп знает, куда слать запросы.

---

## 1. Две «роли» ссылок (это не две разные подписки)

| Роль | Как выглядит | Где задаётся |
|------|-------------|--------------|
| **Upstream (панель)** | `https://95.181.175.67:2096/sub_bitvpn/...` **без** `installid` в `.env` | `HAPP_SUBSCRIPTION_URL` в `.env` на **сервере** |
| **Публичная (пользователю)** | `https://bitecosystem.ru/sub/XXXXXXXXXXXX` (12 символов) | Генерируется ботом/API, если задан `HAPP_SUBSCRIPTION_REDIRECT_BASE` |
| **Старая форма «как раньше»** | `https://95.181...?installid=КОД` | То же самое по данным, что `bitecosystem.ru/sub/КОД`, только адрес «голый» панельный |

**Важно:** контент подписки (строки `vless://` и т.д.) сервер для `/sub/КОД` **скачивает с панели**:  
`HAPP_SUBSCRIPTION_URL` + `?installid=КОД` (или `&installid=`, если в базовом URL уже есть `?`).

Если в `.env` указана **не та** база панели или панель недоступна с VPS — **не будет работать** ни «красивая» ссылка, ни ручная проверка.

---

## 2. Как глазами отличить «старую» ссылку от «новой»

### Считаем **новым** (целевой продакшен)

- Домен **`bitecosystem.ru`** (или ваш актуальный API-домен).
- Путь **`/sub/`** + ровно **12 символов** латиницы/цифр:  
  `https://bitecosystem.ru/sub/yHmESPsZKd76`

### Считаем **старым форматом** (прямо панель, всё ещё может работать, но не цель)

- IP **`95.181.175.67`** и порт **`2096`**, путь вида **`sub_bitvpn/...`**, часто с **`?installid=`** в конце:  
  `https://95.181.175.67:2096/sub_bitvpn/bgmdn1s016p08yfb?installid=...`

**Оба варианта** при правильной настройке отдают **один и тот же тип конфига**, если код `installid` тот же и панель жива. Разница — только **красота и скрытие IP** в адресной строке.

### `happ://add/https://...`

Это **не третья подписка**. Так Happ открывается из мини-аппа: к вашей HTTPS-ссылке дописывается префикс `happ://add/`. Подставляется **тот же** URL, что и для копирования (`https://bitecosystem.ru/sub/...`).

## 3. Что должно быть в `.env` (смысл переменных)

Проверяйте **и на локальном ПК** (для деплоя), и **на сервере** в каталоге бота (часто `/opt/vpn-bot/.env`).

| Переменная | Пример значения | Зачем |
|------------|-----------------|--------|
| `HAPP_SUBSCRIPTION_URL` | `https://95.181.175.67:2096/sub_bitvpn/bgmdn1s016p08yfb` | База подписки **с панели**; код `installid` к ней дописывает система/Happ. |
| `HAPP_SUBSCRIPTION_URLS` | `https://95.181.../sub_bitvpn/...,https://109.107.../sub/...` | Опционально: **несколько** upstream подписок (разные VPS/страны). API склеит их в одну выдачу `/sub/<CODE>`. Если пусто — используется только `HAPP_SUBSCRIPTION_URL`. |
| `HAPP_SUBSCRIPTION_REDIRECT_BASE` | `https://bitecosystem.ru` | Публичные ссылки вида `https://bitecosystem.ru/sub/КОД`. Если **пусто** — пользователю могут выдавать длинный URL с IP панели. |
| `MINIAPP_API_URL` | `https://bitecosystem.ru` | Адрес API для бота/скриптов. |
| `WEBAPP_URL` | `https://bitvpn.vercel.app?api=https://bitecosystem.ru` | Открытие мини-аппа: **страница с Vercel**, запросы к данным на `bitecosystem.ru` через `?api=`. |
| `HAPP_API_URL` | `https://api.happ-proxy.com` | База для **`/api/add-install`** (создание кода установки). Если здесь ошибка — новые ссылки не создаются. |
| `HAPP_LIST_INSTALL_URL` | `https://happ-proxy.com` | Опционально: для **счётчика устройств** (list-install). Если не задан — используется `HAPP_API_URL` (см. код/журнал). |
| `HAPP_PROVIDER_CODE`, `HAPP_AUTH_KEY` | из кабинета Happ | Без них add-install и учёт не работают. |

После **любой** правки `.env` на сервере:

```bash
sudo systemctl restart vpn-bot
sudo systemctl restart miniapp-api
```

## 4. Где ещё прописано (не только `.env`)

| Место | Что проверить |
|-------|----------------|
| **BotFather** → ваш бот → **Menu Button / Mini App URL** | Должно совпадать по смыслу с `WEBAPP_URL`: есть **`?api=https://bitecosystem.ru`**. |
| **Vercel** | Задеплоен актуальный `index.html` (после правок в `webapp/`). |
| **Nginx на 213.165.38.222** | Запросы **`https://bitecosystem.ru/sub/...`** и **`/api/...`** попадают в **miniapp-api** (как в ваших `deploy/*.conf`). |

## 5. Пошаговая проверка с **сервера** (SSH)

Подключитесь (пример):

```bash
ssh root@213.165.38.222
```

### 5.1. Убедиться, что на сервере «тот» `.env`

```bash
grep -E '^HAPP_SUBSCRIPTION_URL=|^HAPP_SUBSCRIPTION_REDIRECT_BASE=|^MINIAPP_API_URL=|^HAPP_API_URL=|^WEBAPP_URL=' /opt/vpn-bot/.env
```

(Путь замените на свой, если бот не в `/opt/vpn-bot`.)

Сверьте: **`HAPP_SUBSCRIPTION_URL`** — ровно база с панели (как в 3x-ui, subscription URL), **без** лишнего `installid` в самой переменной.

### 5.2. Панель отвечает (upstream)

Подставьте **реальный 12-символьный код** из ссылки пользователя (после `/sub/`):

```bash
CODE=yHmESPsZKd76
curl -sS -o /tmp/sub_test.txt -w "HTTP %{http_code}\n" \
  "https://95.181.175.67:2096/sub_bitvpn/bgmdn1s016p08yfb?installid=${CODE}"
head -c 200 /tmp/sub_test.txt; echo
```

Ожидается **HTTP 200** и начало текста подписки (base64 / `vless://` в декоде). Если **000**, **timeout**, **SSL error** — с VPS до панели нет доступа или неверный URL панели.

### 5.3. Публичная ссылка (через ваш API)

```bash
CODE=yHmESPsZKd76
curl -sS -o /tmp/sub_public.txt -w "HTTP %{http_code}\n" \
  "https://bitecosystem.ru/sub/${CODE}"
head -c 200 /tmp/sub_public.txt; echo
```

Ожидается **200** и **похожий** контент на шаг 5.2 (могут отличаться комментарии `#providerid`, имя в конфиге — это нормально).

### 5.4. API жив

```bash
curl -sS -o /dev/null -w "%{http_code}\n" "https://bitecosystem.ru/api/health"
```

Ожидается **200**.

### 5.5. Логи miniapp-api при проблеме с `/sub/`

```bash
journalctl -u miniapp-api -n 100 --no-pager
```

Ищите ошибки `requests`, `sub proxy`, `503`, `timeout`.

## 6. Проверка с **Windows** (PowerShell) — без SSH

Только **публичный** URL (панель с домашнего IP может быть недоступна из-за файрвола):

```powershell
Invoke-WebRequest -Uri "https://bitecosystem.ru/sub/yHmESPsZKd76" -UseBasicParsing | Select-Object StatusCode
```

Статус **200** — хорошо. Если **404/502** — смотреть DNS, SSL, nginx, сервис на сервере.

Проверка, что мини-апп «видит» API (браузер):

- Откройте в браузере:  
  `https://bitecosystem.ru/api/miniapp/plans`  
  Должен вернуться JSON (хотя бы с планами/конфигом), не HTML-ошибка nginx.

## 7. Как понять, что в **базе** ещё «старая» ссылка

Пользователю в приложении отдаётся то, что лежит в **`vpn_config` / подписке** в БД, плюс логика в **`/api/miniapp/me`** может **нормализовать** формат при открытии мини-аппа.

Практично:

1. Открыть мини-апп из бота **после** исправления `.env` и перезапуска.
2. В профиле / «Мой конфиг» запросить ссылку снова.
3. Если везде задан `HAPP_SUBSCRIPTION_REDIRECT_BASE`, в выдаче должна быть **`https://bitecosystem.ru/sub/КОД`**.

Точная проверка БД — через `psql`/админку Neon по вашему `DATABASE_URL` (таблица подписок, поле с конфигом). Это опционально, если API уже отдаёт правильную ссылку.

## 8. Частые сбои и что делать

| Симптом | Куда смотреть |
|---------|----------------|
| `/sub/КОД` отдаёт 503 / «Subscription URL not configured» | На сервере пустой или неверный `HAPP_SUBSCRIPTION_URL`. |
| Панель 200 с сервера, а `bitecosystem.ru/sub/` — 502/404 | nginx, `miniapp-api` не запущен, неверный `server_name`/путь. |
| Новые оплаты без ссылки | `HAPP_API_URL` для `add-install` (часто нужен `https://api.happ-proxy.com`), ключи Happ. |
| «Ошибка загрузки» в мини-аппе | `WEBAPP_URL` с `?api=`, CORS, недоступность `POST /api/miniapp/me`. |
| VPN «не тот IP» в браузере | Режим Happ **Proxy vs TUN**, не формат `https` ссылки. |

## 9. Мини-чеклист «закрыли тему на сегодня»

- [ ] `.env` на **сервере** совпадает по смыслу с локальным (или осознанно отличается — вы знаете почему).
- [ ] `HAPP_SUBSCRIPTION_URL` = актуальная **база** из 3x-ui.
- [ ] `HAPP_SUBSCRIPTION_REDIRECT_BASE=https://bitecosystem.ru` (если хотите только «красивые» ссылки).
- [ ] Перезапущены `vpn-bot` и `miniapp-api`.
- [ ] `curl` с сервера: панель и `https://bitecosystem.ru/sub/КОД` дают **200**.
- [ ] BotFather: URL мини-аппа с **`?api=https://bitecosystem.ru`**.
- [ ] Vercel: задеплоен актуальный фронт после правок.

Если все галочки зелёные — проблема «старая vs новая ссылка» на стороне конфигурации **снята**; остаются только сеть, Happ-клиент и панель Xray.
