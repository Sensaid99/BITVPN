# Настройка Bit VPN бота с нуля

Пошагово: что установить, куда нажать, что прописать, чтобы бот, мини-апп и API работали вместе.

---

## Что должно быть у вас

- **Сервер (VPS)** с Ubuntu, с белым IP и доменом (например `bitecosystem.ru` → IP `213.165.38.222`).
- **Аккаунт Telegram** и созданный бот через @BotFather (токен).
- **База данных** — PostgreSQL (например Neon) или SQLite; строка подключения `DATABASE_URL`.
- **ЮKassa** — магазин, `SHOP_ID` и `SECRET_KEY`.
- **Happ** — кабинет провайдера, `provider_code` (8 символов) и `auth_key` (32 символа); базовая ссылка подписки (из 3x-ui или панели).
- **GitHub** — репозиторий с проектом (у вас Sensaid99/BITVPN).
- **Vercel** — аккаунт для деплоя мини-аппа.

---

## Часть 1. Сервер (Ubuntu)

### 1.1 Подключение и базовая установка

Подключитесь по SSH (подставьте свой IP и пользователя):

```bash
ssh root@213.165.38.222
```

Установите необходимое:

```bash
apt update && apt install -y git python3 python3-pip python3-venv nginx certbot python3-certbot-nginx
```

### 1.2 Код проекта и venv

Если папки `/opt/vpn-bot` ещё нет:

```bash
cd /opt
git clone https://github.com/Sensaid99/BITVPN.git vpn-bot
cd vpn-bot
```

Если папка **уже есть** (как у вас) — просто обновите код и зайдите в неё:

```bash
cd /opt/vpn-bot
git fetch origin
git reset --hard origin/main
```

Затем создайте venv (если его ещё нет) и установите зависимости:

```bash
cd /opt/vpn-bot
python3 -m venv venv
./venv/bin/pip install -r requirements.txt
```

(Если в корне нет `requirements.txt`, используйте `pip install -e .` при наличии `pyproject.toml` или установите зависимости из README проекта.)

### 1.3 Файл .env на сервере

nano /opt/vpn-bot/.env

### 1.4 Сервисы systemd (бот и API)

Бот (читает .env из папки `/opt/vpn-bot` сам, дополнительно ничего не нужно):

```bash
cp /opt/vpn-bot/deploy/vpn-bot.service /etc/systemd/system/
```

Затем API:

```bash
cp /opt/vpn-bot/deploy/miniapp-api.service /etc/systemd/system/
systemctl daemon-reload
systemctl enable vpn-bot miniapp-api
systemctl start vpn-bot miniapp-api
systemctl status vpn-bot miniapp-api
```

Оба сервиса должны быть `active (running)`.

### 1.5 Nginx и HTTPS

Настройте сайт для домена (например `bitecosystem.ru`). Пример конфига:

```bash
nano /etc/nginx/sites-available/vpn-api
```

Содержимое (замените домен на свой):

```nginx
server {
    listen 80;
    server_name bitecosystem.ru;
    location / {
        return 301 https://$host$request_uri;
    }
}
server {
    listen 443 ssl;
    server_name bitecosystem.ru;
    ssl_certificate     /etc/letsencrypt/live/bitecosystem.ru/fullchain.pem;
    ssl_certificate_key /etc/letsencrypt/live/bitecosystem.ru/privkey.pem;

    location /sub/ {
        proxy_pass http://127.0.0.1:8765;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;
    }
    location /api/ {
        proxy_pass http://127.0.0.1:8765;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Host $host;
    }
}
```

Включите сайт и получите сертификат:

```bash
ln -sf /etc/nginx/sites-available/vpn-api /etc/nginx/sites-enabled/
certbot --nginx -d bitecosystem.ru
nginx -t && systemctl reload nginx
```

После этого `https://bitecosystem.ru/api/...` должен проксироваться на API (порт 8765).

## Часть 2. ЮKassa

1. Зайдите в личный кабинет ЮKassa.
2. Настройки → Уведомления (Webhook).
3. URL уведомлений: `https://bitecosystem.ru/api/webhook/yookassa`
4. Включите событие **payment.succeeded**.
5. Сохраните.

Так после оплаты ЮKassa будет вызывать ваш API, и бот сможет выдать ссылку автоматически.

## Часть 3. Мини-апп (Vercel)

1. Зайдите на [vercel.com](https://vercel.com), войдите через GitHub.
2. Add New → Project → выберите репозиторий **Sensaid99/BITVPN** (или свой).
3. **Root Directory** укажите **`webapp`** (чтобы деплоилась только папка с мини-аппом).
4. Deploy. Скопируйте URL вида `https://bitvpn.vercel.app` (или ваш).
5. В `.env` на сервере (и локально) должно быть:
   - `WEBAPP_URL=https://bitvpn.vercel.app?api=https://bitecosystem.ru`  
   Без `?api=...` приложение не будет знать, куда слать запросы (подписки, оплата).

После смены WEBAPP_URL перезапустите бота на сервере:

```bash
systemctl restart vpn-bot
```

## Часть 4. Happ

1. Зайдите на [happ-proxy.com](https://happ-proxy.com) (или happ.su) как **провайдер**.
2. В кабинете возьмите **provider_code** (8 символов) и **auth_key** (32 символа).
3. В `.env` на сервере пропишите:
   - `HAPP_API_URL=https://happ-proxy.com`  
     (если поддержка Happ даст другой URL API — укажите его.)
   - `HAPP_PROVIDER_CODE=...`
   - `HAPP_AUTH_KEY=...`
   - `HAPP_SUBSCRIPTION_URL=...` — базовая ссылка подписки **без** `?installid=...` (обычно из 3x-ui или панели).

4. Перезапустите API:

```bash
systemctl restart miniapp-api
```

Если с сервера запросы к Happ возвращают 404 — уточните в поддержке Happ актуальный **базовый URL API** и при необходимости поменяйте `HAPP_API_URL`.

## Часть 5. Проверки

Сделайте по порядку.

1. **Бот**  
   Напишите боту в Telegram — он должен отвечать. В меню должна быть кнопка «Открыть приложение».

2. **API**  
   В браузере откройте:
   - `https://bitecosystem.ru/api/miniapp/check-happ-env`  
   Должен вернуться JSON с `"ok": true` и всеми `env_set: true`. Если `ok: false` — проверьте переменные в `.env` на сервере.

3. **Мини-апп**  
   В боте нажмите «Открыть приложение». Должна открыться страница с Vercel; подписки и кнопки оплаты подгружаются с вашего API.

4. **Оплата и ссылка**  
   В мини-аппе выберите тариф → способ оплаты → оплатите (тест). После успешной оплаты ЮKassa вызовет вебхук → бот должен прислать сообщение и ссылку Happ. В разделе «Устройства» в мини-аппе при активной подписке должна появляться ссылка и кнопка «Скопировать».

## Часть 6. Деплой после изменений

- **Только бот или API (код на сервере):**  
  Запустите **ДЕПЛОЙ_НА_СЕРВЕР.bat** с ПК (из папки проекта). Скрипт сделает push в GitHub, на сервере выполнит `git pull` и перезапустит бота и API.

- **Только интерфейс мини-аппа:**  
  Запустите **ДЕПЛОЙ_МИНИАПП_VERCEL.bat** или сделайте push в GitHub при подключённом автодеплое Vercel (Root Directory = `webapp`).

- **И бот, и мини-апп:**  
  Сначала деплой на сервер, затем при необходимости деплой мини-аппа на Vercel.
