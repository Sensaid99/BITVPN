# Запуск бота и первый деплой

## 1. Библиотеки

В корне проекта один раз:

```bash
pip install -r requirements.txt
```

## 2. Настройка .env

В `.env` укажите токен бота, админов, при необходимости — способы оплаты (ЮKassa и т.д.) и `WEBAPP_URL` для кнопки мини-аппа.

## 3. Мини-апп на Vercel

- Страница мини-аппа (HTML/JS) деплоится на Vercel → получаете `https://ваш-проект.vercel.app`.
- В `.env`: `WEBAPP_URL=https://ваш-проект.vercel.app` (без слеша в конце).
- Подробно: **MINI_APP_VERCEL.md**, **API_VERCEL_ПОШАГОВО.md** в этой папке.

## 4. Запуск бота локально

```bash
python run.py
```

или `python bot/main.py`.

Демо без токена: `python demo_bot.py`.

## 5. Скрипты деплоя в корне проекта

| Скрипт | Назначение |
|--------|------------|
| **ДЕПЛОЙ_НА_СЕРВЕР.bat** | Синхронизация webapp → public → index/api, пуш в GitHub (ветка из deploy_config), копирование .env (если включено), на сервере: git fetch + checkout + reset --hard + перезапуск бота и miniapp-api. |
| **ДЕПЛОЙ_МИНИАПП_API_ТОЛЬКО.bat** | Синхронизация, пуш в GitHub, деплой папки webapp на Vercel, на сервере обновление кода и перезапуск miniapp-api. |
| **ДЕПЛОЙ_МИНИАПП_VERCEL.bat** | Только деплой мини-аппа (папка webapp) на Vercel. |
| **ДЕПЛОЙ_ВСЁ.bat** | Сначала Vercel (мини-апп), затем полный деплой на сервер в одном окне. |
| **ПУШ_НА_ГИТХАБ.bat** | Синхронизация и пуш в текущую ветку (для редеплоя Vercel без сервера). |

Настройки в **deploy_config.cmd**: `SERVER_USER`, `SERVER_IP`, `BOT_PATH`, `RESTART_CMD`, **GIT_BRANCH** (ветка для пуша и обновления на сервере, по умолчанию `main`), `COPY_ENV_TO_SERVER` (1 = копировать .env на сервер при деплое). Ветка **GIT_BRANCH** должна совпадать с основной веткой в GitHub (main или master). На сервере при обновлении выполняется `git reset --hard origin/%GIT_BRANCH%`, чтобы локальные изменения на сервере не блокировали pull.

- **ЗАПУСТИТЬ_БОТА.bat**, **ЗАПУСТИТЬ_МИНИАПП_API.bat** — локальный запуск.
- Подробный список: **docs/other/КНОПКИ_И_СКРИПТЫ.txt**.
