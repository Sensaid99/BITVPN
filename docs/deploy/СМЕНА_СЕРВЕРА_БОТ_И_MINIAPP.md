# Смена сервера бота и мини‑апп API

Краткий чеклист при переносе на новый VPS/домен.

## 1. DNS и HTTPS

- Выдайте домену A‑запись на новый IP (или CNAME на хостинг).
- Выпустите сертификат (Let's Encrypt / панель хостинга).
- Nginx: прокси на `miniapp-api` (порт из unit‑файла) и при необходимости статику мини‑аппа.

## 2. Переменные окружения на новом сервере (`.env`)

Обновите и проверьте:

| Переменная | Назначение |
|------------|------------|
| `MINIAPP_API_URL` | Базовый URL нового API, **https://ваш-домен** (без слэша в конце). |
| `WEBAPP_URL` | URL мини‑аппа в Telegram: `https://страница-на-vercel-или-статик?api=https://ваш-домен` — параметр `api=` должен указывать на **тот же** хост, что и `MINIAPP_API_URL`. |
| `HAPP_SUBSCRIPTION_REDIRECT_BASE` | Тот же **https://ваш-домен**, что и API, чтобы ссылки были `https://домен/sub/КОД` (валидный SSL, не IP). |
| `BOT_TOKEN`, `DATABASE_URL`, `HAPP_*` | Как на старом сервере (или миграция БД). |

После правок: `sudo systemctl restart miniapp-api` и сервис бота (например `vpn-bot` / `python run.py`).

## 3. Telegram BotFather

- **Web App URL** — URL открытия мини‑аппа (часто Vercel/static), с `?api=https://новый-домен` если так у вас заведено.

## 4. Деплой кода

- Залить репозиторий, установить зависимости, перезапустить процессы.
- Убедиться, что API жив: `GET https://ваш-домен/api/health` или `GET https://ваш-домен/api/miniapp/plans` (корень `/health` за nginx без отдельного `location` часто даёт **404** — проксируются только `/api/` и `/sub/`).

## 5. Проверка

- Открыть мини‑апп из бота → данные профиля/подписки грузятся.
- В Happ ссылка «для настройки» — **домен**, не IP (при корректном `HAPP_SUBSCRIPTION_REDIRECT_BASE`).
