# Одна подписка — несколько серверов (Happ / 3x-ui)

**Цель:** пользователь открывает **одну** ссылку подписки (как сейчас через `bitecosystem.ru/sub/...` и `HAPP_SUBSCRIPTION_URL`), а в Happ видит **несколько серверов** (например Нидерланды + Финляндия).

**Важно:** список серверов формирует **3x-ui** в ответе на URL подписки. Бот и `.env` **не обязаны меняться**, пока **базовый** URL подписки остаётся тем же (`HAPP_SUBSCRIPTION_URL` на **основной** панели).

---

## Какая панель «главная»

| Роль | Обычно |
|------|--------|
| **Мастер (основная)** | Тот VPS, чей `https://.../sub_.../...` стоит в **`HAPP_SUBSCRIPTION_URL`** (у вас панель **95.181.175.67**). |
| **Вторая нода** | Новый VPS (**109.107.190.148**, Финляндия) — отдельная 3x-ui, пока со **своей** ссылкой подписки. |

Чтобы была **одна** подписка — ноду Финляндии нужно **подключить к мастеру** или включить её inbound’ы в **общий** ответ подписки мастера.

---

## Способ 1 — Ноды (Nodes / Multi-node) в 3x-ui — предпочтительно

В новых ветках **3x-ui** есть режим **удалённой ноды**: мастер на **95.181** управляет или подтягивает конфиги с **второго** сервера, а **ссылка подписки мастера** отдаёт **несколько** `vless://` (по одному на ноду).

### На сервере Финляндии (дочерняя панель)

1. Зайдите в **3x-ui** → **Настройки панели** (или **Panel Settings**).
2. Найдите раздел про **Node** / **API** / **Remote** (название зависит от версии):
   - включите **режим ноды** или **доступ API для мастера**;
   - задайте **секрет / API key**;
   - при необходимости укажите **доверенный IP** мастера (**95.181.175.67**), если панель это спрашивает.
3. Убедитесь, что **inbound** на 443 (Reality и т.д.) **включён** и клиенты созданы.

### На мастере (95.181)

1. **Настройки** → **Ноды** / **Nodes** / **Manage nodes** (если пункта нет — обновите 3x-ui до версии с multi-node или см. способ 2).
2. **Добавить ноду**:
   - **Адрес API** дочерней панели: например `https://109.107.190.148:2053/...` (ваш путь к панели, как в браузере, без лишнего).
   - **Ключ / токен** — тот, что задали на Финляндии.
3. Сохраните, дождитесь статуса **онлайн / OK**.
4. В настройках ноды или подписки выберите, **какие inbound’ы** с ноды попадают в **общую** подписку (если интерфейс это предлагает).

### Проверка

1. Откройте в браузере **старую** базовую ссылку подписки мастера (ту, что в `HAPP_SUBSCRIPTION_URL`, **без** `installid`):
   - в тексте должно быть **две (и более)** строки `vless://...` с **разными** хостами/IP (95.181… и 109.107…).
2. Если по-прежнему одна строка — в панели мастера ещё не включили выдачу второй ноды в подписку (см. документацию к **вашей** версии 3x-ui или issue на GitHub проекта [MHSanaei/3x-ui](https://github.com/MHSanaei/3x-ui)).

## Способ 2 — Два inbound на **одной** панели (оба «сервера» без отдельной ноды)

Подходит, если **оба** протокола слушаются на **одном** IP (один VPS).  
**Не ваш случай**, если Финляндия — **отдельный** IP: второй «сервер» в Happ будет с тем же IP, только другой порт/имя — гео не станет «Финляндия».

Имеет смысл только как **два разных протокола/порта** на одном железе.

---

## Способ 3 — Вручную склеить подписку (временный костыль)

1. Скачайте текст с `https://95.181.../sub...` и с `https://109.107...:2096/sub/...`.
2. Декодируйте base64 (если нужно), **склейте** все строки `vless://` в **один** файл, снова закодируйте в base64 (как отдаёт панель).
3. Отдайте этот файл по **одному** URL (nginx/статика на **bitecosystem.ru**) и подставьте этот URL в `HAPP_SUBSCRIPTION_URL`.

Минусы: при смене ключей Reality нужно **вручную** обновлять файл; Happ API / installid логичнее вести с **одной** панели. Для продакшена лучше **способ 1**.

---

## После того как в одной ссылке две ноды

1. **`.env`:** `HAPP_SUBSCRIPTION_URL` остаётся базой **мастера** (95.181), **не** финской панели (если только вы не перевели мастер на другой сервер осознанно).
2. Пользователи в Happ: **обновить подписку** (потянуть вниз / кнопка обновления).
3. В списке серверов появятся **две** записи — можно выбрать Финляндию или старый узел.

---

## Если в меню нет «Ноды»

- Обновите **3x-ui** на мастере и на Финляндии до **одной линейки версий** (см. релизы [3x-ui](https://github.com/MHSanaei/3x-ui/releases)).
- Или временно держите **две подписки** в Happ, пока не настроите ноды.
