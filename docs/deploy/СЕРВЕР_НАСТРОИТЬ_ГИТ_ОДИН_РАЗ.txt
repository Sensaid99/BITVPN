  НА СЕРВЕРЕ ПАПКА НЕ РЕПОЗИТОРИЙ — ОДИН РАЗ НАСТРОИТЬ GIT
═══════════════════════════════════════════════════════════════════

Если при деплое видите: "fatal: not a git repository" и "git pull не сработал",
значит папка с ботом на сервере не клонирована из GitHub. Выполните один раз:

■ 1. Подключитесь к серверу по SSH
    ssh root@155.212.164.135
    (введите пароль)

■ 2. Перейдите в папку бота
    cd /opt/vpn-bot

■ 3. Сделайте папку репозиторием и подтяните код с GitHub
    Важно: перед reset сохраните .env — после reset он может пропасть (его нет в репо):
    cp .env .env.bak

    git init
    git remote add origin https://github.com/Sensaid99/BITVPN.git
    git fetch origin main
    git branch -M main
    git reset --hard origin/main

    (У репозитория Sensaid99/BITVPN основная ветка — main. Если у вас ветка master, замените main на master в трёх строках выше.)

■ 4. Восстановите .env (после reset --hard его часто нет в папке)
    cp .env.bak .env
    Если .env.bak не делали — скопируйте .env с ПК: см. СЕРВЕР_ПОСЛЕ_ГИТ_ВОССТАНОВИТЬ.txt
    Важно: в .env должна быть строка WEBAPP_URL=https://bitvpn.vercel.app (иначе кнопка даст 404 — см. ПРОВЕРКА_ПОСЛЕ_ДЕПЛОЯ.md).

■ 5. Привяжите ветку к origin (чтобы git pull работал без указания ветки)
    git branch --set-upstream-to=origin/main main

■ 6. Перезапустите бота
    sudo systemctl restart vpn-bot

После этого деплой с ПК (ДЕПЛОЙ_НА_СЕРВЕР.bat или Запустить_деплой_на_сервер.vbs) будет обновлять код на сервере через git pull.

■ Если вы уже настроили по инструкции с веткой master, а на GitHub ветка main
  Подключитесь по SSH (cd /opt/vpn-bot) и выполните:
    cp .env .env.bak
    git fetch origin main
    git branch -M main
    git reset --hard origin/main
    cp .env.bak .env
    git branch --set-upstream-to=origin/main main
    sudo systemctl restart vpn-botimage.png
  Затем с ПК скопируйте .env на сервер (СКОПИРОВАТЬ_ENV_НА_СЕРВЕР.bat) и снова перезапустите бота.
