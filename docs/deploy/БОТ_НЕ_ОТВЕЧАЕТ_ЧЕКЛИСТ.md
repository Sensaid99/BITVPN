# Бот «не открывается» / не отвечает на Старт — чеклист

## 0. Бот «молчит» на всё: что проверить по порядку

### 0.1. Переменная токена в `.env` на сервере

Основной бот читает **`BOT_TOKEN`**, а не `HELPBIT_BOT_TOKEN` и не имя из другого сервиса.

```bash
grep -E '^BOT_TOKEN=|^HELPBIT' /opt/vpn-bot/.env
```

Должна быть строка **`BOT_TOKEN=число:секрет`**. Если токен только под другим именем — бот либо не стартует, либо в `systemd` подставлен другой env.

Проверка, что Python реально видит токен из каталога бота:

```bash
cd /opt/vpn-bot && ./venv/bin/python -c "from bot.config.settings import Config; t=Config.BOT_TOKEN; print('OK' if t and ':' in t else 'FAIL', len(t or ''))"
```

### 0.2. Один процесс polling на токен

Два процесса с **одним** `BOT_TOKEN` (два `run.py`, бот + тест у себя на ПК) — Telegram отдаёт апдейты **то одному, то другому**; кажется, что бот «не отвечает».

```bash
pgrep -af 'run.py|bot.main' 
sudo ss -tlnp | grep -E 'python|443'
```

Должен быть **один** рабочий процесс `vpn-bot` на сервере.

### 0.3. Команда `/ping` (без БД)

После деплоя с актуальным кодом в чат с ботом отправьте **`/ping`**. Ожидается ответ **`🏓 pong`** и в логе строка **`ping_command: reply ok`**.

- **`/ping` не отвечает** — проблема до логики `/start`: сеть до `api.telegram.org`, блокировка исходящих, неверный токен, второй poller, **Forbidden** (пользователь в чёрном списке у бота — редко).
- **`/ping` работает, `/start` нет** — смотрите логи **`start_command: sending reply`**, **`reply_text ok`**, ошибки **`BadRequest`**.

### 0.4. Исходящие к Telegram с VPS

```bash
curl -sS -o /dev/null -w '%{http_code}\n' --connect-timeout 5 https://api.telegram.org/
```

Должен быть **`200`** или **`302`**, не таймаут.

### 0.5. Webhook (для режима polling должен быть пустой)

См. раздел 1 ниже.

## 1. В логе есть `start_command`?

```bash
sudo journalctl -u vpn-bot -f
```

Нажмите **Старт** в чате с ботом.

- **Строки `start_command` нет** — апдейт не доходит: другой процесс с тем же токеном, **webhook** вместо polling, бот заблокирован у пользователя.
- **Строка есть** — смотрите ошибки ниже по логу (`ERROR`, `Exception`).

Проверка webhook (должен быть пустой `url`):

```bash
cd /opt/vpn-bot && source venv/bin/activate
python -c "from bot.config.settings import Config; import httpx; print(httpx.get(f'https://api.telegram.org/bot{Config.BOT_TOKEN}/getWebhookInfo').json())"
```

## 2. Happ API и админ

Раньше **`https://happ-proxy.com/api/add-install`** давал **404**. Нужно **`HAPP_API_URL=https://api.happ-proxy.com`** в `.env` (или то, что даёт поддержка Happ).

Синхронный запрос к Happ **не должен блокировать** ответы другим пользователям — в коде вызовы вынесены в поток (`asyncio.to_thread`).

## 3. Один `DATABASE_URL`

У **`vpn-bot`** и **`miniapp-api`** на сервере одинаковая строка подключения к PostgreSQL.

## 4. Мини-приложение «не открывается»

Это отдельно от `/start`: проверьте **`WEBAPP_URL`**, **`MINIAPP_API_URL`**, **`BOT_TOKEN`** у API и рестарт **`miniapp-api`**. См. **СИНХРОНИЗАЦИЯ_БОТ_МИНИАПП.md**.

## 5. После правок `.env`

```bash
sudo systemctl restart vpn-bot miniapp-api
```
