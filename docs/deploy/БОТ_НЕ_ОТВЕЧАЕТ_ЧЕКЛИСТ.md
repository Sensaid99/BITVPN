# Бот «не открывается» / не отвечает на Старт — чеклист

## 1. В логе есть `start_command`?

```bash
sudo journalctl -u vpn-bot -f
```

Нажмите **Старт** в чате с ботом.

- **Строки `start_command` нет** — апдейт не доходит: другой процесс с тем же токеном, **webhook** вместо polling, бот заблокирован у пользователя.
- **Строка есть** — смотрите ошибки ниже по логу (`ERROR`, `Exception`).

Проверка webhook (должен быть пустой `url`):

```bash
cd /opt/vpn-bot && source venv/bin/activate
python -c "from bot.config.settings import Config; import httpx; print(httpx.get(f'https://api.telegram.org/bot{Config.BOT_TOKEN}/getWebhookInfo').json())"
```

## 2. Happ API и админ

Раньше **`https://happ-proxy.com/api/add-install`** давал **404**. Нужно **`HAPP_API_URL=https://api.happ-proxy.com`** в `.env` (или то, что даёт поддержка Happ).

Синхронный запрос к Happ **не должен блокировать** ответы другим пользователям — в коде вызовы вынесены в поток (`asyncio.to_thread`).

## 3. Один `DATABASE_URL`

У **`vpn-bot`** и **`miniapp-api`** на сервере одинаковая строка подключения к PostgreSQL.

## 4. Мини-приложение «не открывается»

Это отдельно от `/start`: проверьте **`WEBAPP_URL`**, **`MINIAPP_API_URL`**, **`BOT_TOKEN`** у API и рестарт **`miniapp-api`**. См. **СИНХРОНИЗАЦИЯ_БОТ_МИНИАПП.md**.

## 5. После правок `.env`

```bash
sudo systemctl restart vpn-bot miniapp-api
```
