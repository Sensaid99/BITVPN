# Кнопка обновления подписки в Happ (и других клиентах)

## Что делает кнопка

В Happ (иконка **круговых стрелок** в шапке) — это **повторная загрузка** того же **URL подписки**, который вы добавили в приложение. Код клиента мы не меняем: приложение просто делает **новый HTTP GET** по этому адресу и подставляет полученный список нод.

С нашей стороны важно:

1. **Сервер отдаёт актуальный список** при каждом запросе (без устаревшего кэша на стороне API).
2. **После добавления сервера в 3x-ui** этот сервер должен попасть в **ту же** цепочку upstream, из которой собирается подписка для пользователя (см. `HAPP_SUBSCRIPTION_URL` / `HAPP_SUBSCRIPTION_URLS` и **ОДНА_ПОДПИСКА_НЕСКОЛЬКО_СЕРВЕРОВ_3XUI.md**).

## Что уже делает наш `GET /sub/<КОД>`

- Проксирует подписку с 3x-ui с **фиксированным User-Agent** (`HAPP_SUBSCRIPTION_FETCH_UA` при необходимости), чтобы на телефоне и ПК был **одинаковый** набор нод.
- Склеивает ноды с нескольких upstream URL и убирает дубликаты.
- Отдаёт ответ с заголовками **без кэширования** (`Cache-Control`, `Pragma`, `Expires`), чтобы клиент не показывал старый конфиг.

После изменений в панели пользователю достаточно **нажать обновление в Happ** — подтянется свежий ответ.

## Если добавили новый сервер, а в Happ его нет

1. В **3x-ui** новый inbound/сервер должен участвовать в **той же подписке** (subscription), что и остальные, **или** вы добавили второй базовый URL в **`HAPP_SUBSCRIPTION_URLS`** (через запятую/как у вас в `.env`) и перезапустили **miniapp-api**.
2. Убедитесь, что ссылка пользователя ведёт на **ваш** домен **`https://…/sub/КОД`**, а не на прямой старый URL панели без прокси.
3. В Happ нажмите **обновление**; при необходимости удалите подписку и добавьте ссылку заново (редко нужно).

## Красивые имена серверов в списке Happ

Если в `.env` заданы **два** URL в **`HAPP_SUBSCRIPTION_URLS`**, подписи к нодам задаются по порядку:

```env
HAPP_SUBSCRIPTION_NODE_NAMES=Wi‑Fi Нидерланды,Wi‑Fi Финляндия
```

Порядок **совпадает** с порядком URL. Без запятой внутри названия (или задайте одно общее имя через **`SUBSCRIPTION_DISPLAY_NAME`**). После правки — **`sudo systemctl restart miniapp-api`**, в Happ — обновить подписку.

## Почему на телефоне 1 сервер, а на ПК 2

Частая причина — панель **3x-ui** отдаёт **разный** состав нод в зависимости от **User-Agent**. Happ на телефоне шлёт «мобильный» UA. В коде API запрос к upstream идёт с **фиксированным десктопным UA**; при странном поведении панели задайте **`HAPP_SUBSCRIPTION_FETCH_UA`** в `.env` (см. `.env.example`). Другая причина — в **`HAPP_SUBSCRIPTION_URLS`** реально указан **один** URL или второй панельный URL отдаёт пусто/ошибку (смотрите **`/api/miniapp/debug-sub-content`** при включённой отладке).

## Итог

**«Сделать кнопку» в коде Happ нельзя** — она уже есть. Нужно, чтобы **URL подписки** вёл на наш прокси и чтобы **upstream** в `.env`/панели отражал все серверы. Тогда после добавления ноды в панели достаточно **обновить подписку в приложении**.
