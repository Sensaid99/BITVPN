# Зашифрованные ссылки Happ (`happ://crypt*`)

## Включение

В `.env` на сервере и локально (для бота):

```env
HAPP_ENCRYPT_SUBSCRIPTION_LINKS=true
```

Перезапустите **miniapp-api** и **vpn-bot**.

## Поведение

- В **мини-апп** (`/api/miniapp/me`) в поле `subscription_link` отдаётся **happ://crypt…** вместо `https://…/sub/…` (если crypto API ответил успешно).
- В **боте** (карточка «Моя подписка», файл `.txt`, fallback «Подключиться») пользователь тоже видит **happ://crypt…**.
- В **БД** по-прежнему хранится **HTTPS**-ссылка — по ней считается **счётчик устройств** (Happ `list-install` + `install_code` из `installid=` или `/sub/CODE`).

## Счётчик устройств

И в боте, и в мини-аппе используется один и тот же механизм: `install_code` из **исходной** HTTPS-ссылки, запрос к `HAPP_LIST_INSTALL_URL` или `HAPP_API_URL`. Если в ответе пусто — показывается лимит из тарифа и подсказка в мини-аппе.

## API шифрования

Используется `POST https://crypto.happ.su/api-v2.php` (см. `happ_client.encrypt_subscription_url_to_crypto`).
