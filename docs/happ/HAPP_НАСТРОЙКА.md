# Настройка выдачи VPN через Happ (Happ-Proxy)

Бот уже умеет выдавать лимитированные ссылки для приложения **Happ** после оплаты. Вам не нужен свой VPN-сервер — нужен только аккаунт **провайдера** на платформе Happ и подписка (прокси/VPN), которую вы раздаёте.

---

## Где взять Happ и что это

- **Happ** (Happ-Proxy) — платформа для раздачи подписок в приложение [Happ](https://www.happ.su/) (прокси/VPN для пользователей).
- **Вы как провайдер** регистрируетесь, получаете доступ к API и даёте боту ключи. После оплаты бот создаёт лимитированную ссылку через API Happ и отправляет её пользователю (ссылка вставляется в приложение Happ).

Документация API: [docs.happ-proxy.com](https://docs.happ-proxy.com/ru/getting-started/api)

---

## Шаг 1. Стать провайдером на Happ

1. Зайдите на [happ.su](https://www.happ.su/) или [happ-proxy.com](https://happ-proxy.com).
2. Зарегистрируйтесь / войдите как **провайдер** (не как обычный пользователь).
3. В личном кабинете провайдера найдите:
   - **provider_code** — 8 символов (латиница/цифры), ID провайдера.
   - **auth_key** — ключ авторизации API (32 символа).

Если раздел «API» или «Для разработчиков» не виден — напишите в поддержку Happ, что нужен доступ к API для выдачи лимитированных ссылок.

---

## Шаг 2. Подписка (ссылка на вашу подписку)

Бот к каждой ссылке подставляет параметр `installid=КОД`. Пользователь открывает ссылку вида:

`https://ваш-сервер-подписки.example.com/sub?installid=ABCDEF123456`

Вам нужна **базовая ссылка** на вашу подписку (тот URL, куда пользователь переходит, чтобы подтянуть подписку в приложение Happ). Обычно это ссылка на ваш прокси/VPN-сервер или панель, которая отдаёт конфиг подписки.

- Если подписку отдаёт сам Happ по вашим настройкам — в кабинете провайдера будет указан ваш URL подписки.
- Если подписку отдаёт ваш сервер или другая панель — используйте этот URL (без параметра `installid`).

Пример: `https://sub.yourvpn.com/me` или то, что даёт вам Happ для провайдера.

---

## Шаг 3. Привязать домен подписки в Happ (если нужно)

По [документации](https://docs.happ-proxy.com/ru/getting-started/api) для работы лимитированных ссылок домен подписки нужно привязать к вашему provider id через API:

- Эндпоинт: `GET /api/add-domain`
- Параметры: `provider_code`, `auth_key`, `domain_hash` (SHA-256 от домена подписки в нижнем регистре, 64 hex-символа), при желании `domain_name`.

Сделать это можно один раз (скриптом или вручную через curl), см. примеры в документации.

---

## Шаг 4. Прописать переменные в .env

В папке проекта откройте файл **.env** и заполните (без кавычек, без пробелов вокруг `=`):

```env
HAPP_API_URL=https://api.happ-proxy.com
HAPP_PROVIDER_CODE=ваш_8_символов
HAPP_AUTH_KEY=ваш_32_символа
HAPP_SUBSCRIPTION_URL=https://ваша-базовая-ссылка-подписки
HAPP_SUBSCRIPTION_REDIRECT_BASE=https://ваш-домен-api
```

- **HAPP_API_URL** — используйте `https://api.happ-proxy.com` (основной сайт happ-proxy.com отдаёт 404 для API).
- **HAPP_PROVIDER_CODE** — ваш provider_code из кабинета Happ (8 символов).
- **HAPP_AUTH_KEY** — ваш auth_key из кабинета Happ (32 символа).
- **HAPP_SUBSCRIPTION_URL** — базовая ссылка подписки (то, что вы раздаёте; бот добавит к ней `?installid=...` или выдаст редирект-ссылку).
- **HAPP_SUBSCRIPTION_REDIRECT_BASE** — (необязательно) если задан, пользователь получает ссылку вида `https://этот-домен/sub/КОД` вместо реального URL подписки — так нельзя «обойти» лимит, удалив `installid`. Подробнее: [HAPP_ЗАЩИТА_ССЫЛКИ.md](HAPP_ЗАЩИТА_ССЫЛКИ.md).

Сохраните файл и **перезапустите бота**.

## Как это работает в боте

1. Пользователь оплачивает подписку (платежи нужно настроить отдельно: ЮMoney, QIWI и т.д.).
2. Бот вызывает API Happ: `GET /api/add-install` с параметрами `provider_code`, `auth_key`, `install_limit` (число устройств по тарифу), при желании `note=tg{user_id}`.
3. Happ возвращает `install_code`.
4. Бот формирует ссылку: `HAPP_SUBSCRIPTION_URL` + `?installid=` + `install_code` (или `&installid=`, если в URL уже есть `?`).
5. Бот отправляет пользователю эту ссылку и при необходимости файл .txt с ней. Пользователь вставляет ссылку в приложение Happ.

Число устройств (`install_limit`) бот берёт из типа тарифа (например, `1_month_1` → 1 устройство, `6_months_3` → 3 устройства).

## Имя и описание в приложении (как у LastdepVPN, без домена)

Чтобы в Happ у всех отображалось **название VPN** (например, «BIT VPN») вместо IP, и многострочное **описание** (поддержка, бот, тарифы):

1. Включите редирект: в `.env` задайте **HAPP_SUBSCRIPTION_REDIRECT_BASE** (ваш API: домен или IP, например `https://155.212.164.135`). Тогда пользователи получают ссылку вида `https://ваш-API/sub/КОД`; подключение остаётся к серверу подписки (95.181.175.67).
2. В том же `.env` добавьте:
   - **SUBSCRIPTION_DISPLAY_NAME** — имя, которое увидят все (например `BIT VPN`).
   - **SUBSCRIPTION_DESCRIPTION** — текст под описанием (поддержка, бот, тарифы). Для новой строки в .env пишите `\n`, например:
     ```env
     SUBSCRIPTION_DESCRIPTION=ЕСЛИ НЕ РАБОТАЕТ - ОБНОВИ ПОДПИСКУ\nили напиши в поддержку @bitvpn_support\nБот @YourBotName_bot
     ```
3. Перезапустите **miniapp-api** (и бота). Новые подписки, добавленные по ссылке `/sub/КОД`, будут показывать заданные имя и описание.

Домена для подписки не нужно — подмена делается в нашем API при прокси контента.

---

## Проверка

После заполнения .env и перезапуска бота при успешной оплате пользователь должен получить сообщение со ссылкой и файл с этой ссылкой. Если ссылка не приходит — смотрите логи бота (ошибки вызова Happ API логируются).

Если Happ возвращает ошибку авторизации — проверьте **HAPP_PROVIDER_CODE** и **HAPP_AUTH_KEY** (без лишних пробелов, точное совпадение с кабинетом).
