  Деплой и перезапуск на сервере
═══════════════════════════════════════════════════════════════

  Запуск деплоя: дважды кликните  Запустить_деплой_на_сервер.vbs
  (откроется окно cmd и выполнится деплой; .vbs не открывается в редакторе)

  Редеплой только на Vercel: дважды кликните  Запустить_пуш_на_гитхаб.vbs
  (синхронизация Mini App + пуш на GitHub; Vercel подхватит и задеплоит)

  ДЕПЛОЙ_НА_СЕРВЕР.bat  — полный деплой:
    1. Синхронизация Mini App (public → index.html, api)
    2. Пуш на GitHub
    3. Копирование .env на сервер
    4. По SSH: git pull на сервере и перезапуск (RESTART_CMD)

  ПУШ_НА_ГИТХАБ.bat  — только пуш на GitHub (редиплой Vercel, без сервера).

  ПЕРЕЗАПУСК_НА_СЕРВЕРЕ.bat  — только перезапуск сервисов по SSH
    (без пуша и копирования; использует RESTART_CMD из конфига)

  Перед первым запуском создайте  deploy_config.cmd  в папке проекта:

    set SERVER_USER=root
    set SERVER_IP=ваш_ip_или_домен
    set BOT_PATH=/opt/vpn-bot
    set RESTART_CMD=sudo systemctl restart vpn-bot

  Несколько сервисов (бот + API мини-апп):
    set RESTART_CMD=sudo systemctl restart vpn-bot && sudo systemctl restart miniapp-api

  Если на сервере папка ещё не репозиторий — один раз настройте Git по
  СЕРВЕР_НАСТРОИТЬ_ГИТ_ОДИН_РАЗ.txt

  Если и .vbs не открывает окно: откройте cmd, перейдите в папку проекта
  (cd /d "d:\VPN BOT") и введите:  ДЕПЛОЙ_НА_СЕРВЕР.bat
