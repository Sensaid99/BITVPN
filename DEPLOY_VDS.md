# Развёртывание двух ботов на сервере — пошагово

На сервере будут работать **VPN-бот** и **бот поддержки**. Сервер: **155.212.164.135**, Ubuntu 24.04.

Выполняйте шаги по порядку. Важно: часть шагов — **на вашем компьютере**, часть — **на сервере** (после подключения по SSH). Где что делать, написано в каждом шаге.

---

## Шаг 1. Подключиться к серверу

**Где:** на вашем компьютере.

**Что сделать:**

1. Откройте **PowerShell** (Пуск → введите «PowerShell» → откройте).
2. Введите команду и нажмите Enter:

```
ssh root@155.212.164.135
```

3. Когда появится строка `root@155.212.164.135's password:` — введите пароль от root (тот, что в панели FirstVDS). Пароль при вводе не отображается — это нормально, введите и нажмите Enter.

**Что вы увидите:** приглашение вида `root@nikolay:~#`. Это значит, вы уже **на сервере**. Все следующие команды до шага 2 вводите здесь.

**Дальше:** переходите к шагу 2 (он тоже на сервере).

---

## Шаг 2. Установить Python на сервере

**Где:** сервер (вы уже подключены по SSH).

**Что сделать:** введите по очереди три команды (после каждой нажимайте Enter). Может попросить подтверждение — введите `y` и Enter.

```
apt update
```

*(обновляет список пакетов — подождите окончания)*

```
apt install -y python3 python3-venv python3-pip
```

*(устанавливает Python и инструменты — подождите окончания)*

**Что вы увидите:** в конце строки типа «Setting up ... Done» без красных ошибок.

**Дальше:** шаг 3 — снова на сервере (очистка и подготовка папки).

---

## Шаг 3. Очистить старые файлы и подготовить папку на сервере

**Где:** сервер.

**Зачем:** чтобы в `/opt/vpn-bot` не осталось старых или лишних файлов.

**Что сделать:** введите по очереди:

```
systemctl stop vpn-bot support-bot 2>/dev/null
```

*(останавливает боты, если они уже были запущены — ошибку можно игнорировать)*

```
rm -rf /opt/vpn-bot /opt/vpn-bot.tar.gz
```

*(удаляет старую папку проекта и старый архив)*

```
mkdir -p /opt/vpn-bot
```

*(создаёт пустую папку для проекта)*

**Дальше:** шаг 4 — переключаемся на **ваш компьютер** (без выхода из SSH можно просто открыть второе окно PowerShell).

---

## Шаг 4. Создать архив проекта на вашем компьютере

**Где:** ваш компьютер (Windows). **Не в окне SSH.**

**Важно:** откройте **новое** окно PowerShell (Пуск → PowerShell). В нём не должно быть подключения к серверу (не должно быть приглашения `root@nikolay`).

**Что сделать:**

1. Перейдите в папку проекта. Введите и нажмите Enter:

```
cd "d:\VPN BOT"
```

2. Создайте архив. Введите одну длинную строку целиком и нажмите Enter:

```
tar --exclude=venv --exclude=__pycache__ --exclude=*.db --exclude=.git -czvf vpn-bot.tar.gz .
```

*(точка в конце обязательна)*

Появится список файлов — это нормально.

3. Проверьте размер архива:

```
dir vpn-bot.tar.gz
```

**Что вы увидите:** размер файла должен быть **не меньше примерно 100–200 KB** (лучше больше). Если размер несколько килобайт — вы не в папке с ботом, проверьте путь `d:\VPN BOT`.

**Дальше:** шаг 5 — загрузка этого архива на сервер (всё ещё на ПК).

---

## Шаг 5. Загрузить архив на сервер с вашего компьютера

**Где:** ваш компьютер, то же окно PowerShell, где вы создали архив (папка `d:\VPN BOT`).

**Что сделать:** введите команду и нажмите Enter:

```
scp vpn-bot.tar.gz root@155.212.164.135:/opt/
```

Когда появится запрос пароля — введите пароль root и нажмите Enter.

**Что вы увидите:** строка вида `vpn-bot.tar.gz 100% ...` — значит, файл загружен.

**Дальше:** шаг 6 — снова **на сервере**. Вернитесь в окно, где вы подключены по SSH (где было `root@nikolay:~#`), или подключитесь заново командой `ssh root@155.212.164.135`.

## Шаг 6. Распаковать архив на сервере

**Где:** сервер (окно с `root@nikolay:~#` или после подключения по SSH).

**Что сделать:** введите по очереди:

```
mkdir -p /opt/vpn-bot
```

*(на случай, если папки нет)*

```
tar -xzvf /opt/vpn-bot.tar.gz -C /opt/vpn-bot
```

*(распаковывает архив в папку проекта)*

Появится список распакованных файлов — это нормально.

3. Проверьте, что проект на месте:

```
ls /opt/vpn-bot
```

**Что вы увидите:** в списке должны быть файлы `run.py`, `requirements.txt` и папки `bot`, `support_bot`, `deploy`. Если их нет — значит, архив был создан не из папки `d:\VPN BOT`; повторите шаги 4 и 5 на ПК.

**Дальше:** шаг 7 — создание виртуального окружения (всё на сервере).

---

## Шаг 7. Создать виртуальное окружение и установить зависимости

**Где:** сервер.

**Что сделать:** вводите команды по одной, после каждой нажимайте Enter. Установка может занять 1–2 минуты.

```
cd /opt/vpn-bot
```

```
python3 -m venv venv
```

*(создаёт изолированное окружение Python)*

```
./venv/bin/pip install --upgrade pip
```

*(обновляет pip)*

```
./venv/bin/pip install -r requirements.txt
```

*(устанавливает зависимости VPN-бота — дождитесь окончания)*

```
./venv/bin/pip install -r support_bot/requirements.txt
```

*(устанавливает зависимости бота поддержки)*

**Что вы увидите:** в конце строки «Successfully installed ...». Красные предупреждения о версиях можно игнорировать, если установка завершилась.

**Дальше:** шаг 8 — создать файл с настройками (.env).

---

## Шаг 8. Создать файл .env на сервере

**Где:** сервер.

**Зачем:** в нём хранятся токены ботов и настройки. Без него боты не запустятся.

**Что сделать:**

1. Откройте редактор на сервере:

```
nano /opt/vpn-bot/.env
```

2. На **вашем компьютере** откройте файл `d:\VPN BOT\.env` в Блокноте (или в Cursor). Выделите весь текст (Ctrl+A) и скопируйте (Ctrl+C).

3. В окне SSH (где открыт nano) **вставьте** скопированный текст: правая кнопка мыши в окне терминала или Shift+Insert.

4. Сохраните файл в nano:
   - Нажмите **Ctrl+O** (буква O),
   - Нажмите **Enter** (подтвердить имя файла),
   - Нажмите **Ctrl+X** (выйти из редактора).

5. Ограничьте доступ к файлу (введите в терминале на сервере):

```
chmod 600 /opt/vpn-bot/.env
```

**Дальше:** шаг 9 — включить ботов как сервисы.

---

## Шаг 9. Включить оба бота как сервисы

**Где:** сервер.

**Что сделать:** введите по очереди (каждую строку — Enter):

```
cp /opt/vpn-bot/deploy/vpn-bot.service /etc/systemd/system/
```

```
cp /opt/vpn-bot/deploy/support-bot.service /etc/systemd/system/
```

```
systemctl daemon-reload
```

```
systemctl enable vpn-bot support-bot
```

```
systemctl start vpn-bot support-bot
```

**Что вы увидите:** после `start` ошибок быть не должно.

**Дальше:** шаг 10 — проверить, что оба бота работают.

---

## Шаг 10. Проверить, что боты запущены

**Где:** сервер.

**Что сделать:**

1. Проверьте статус VPN-бота:

```
systemctl status vpn-bot
```

Должна быть строка **Active: active (running)** зелёным. Чтобы выйти из просмотра статуса — нажмите клавишу **Q**.

2. Проверьте статус бота поддержки:

```
systemctl status support-bot
```

Тоже должно быть **Active: active (running)**. Выход — клавиша **Q**.

**Если оба в состоянии running — развёртывание завершено.** Проверьте ботов в Telegram: напишите VPN-боту и боту поддержки — они должны отвечать.

## Как вносить изменения и обновлять ботов на сервере

Сервер заново разворачивать не нужно. Перезагружать весь сервер тоже не нужно — достаточно перезапустить нужный бот (1–2 секунды).

### Обычный цикл: правка → загрузка → перезапуск

1. **На вашем ПК** правите код в папке `d:\VPN BOT` (логика бота, мини-приложение, тексты и т.д.). При желании тестируете локально (запуск `run.py` или бота поддержки у себя).
2. **Загружаете обновлённые файлы на сервер** (см. ниже).
3. **На сервере** перезапускаете только нужный сервис:
   - меняли VPN-бота или общие файлы → `systemctl restart vpn-bot`
   - меняли только бота поддержки → `systemctl restart support-bot`
   - меняли и то и другое → `systemctl restart vpn-bot support-bot`
4. Проверяете в Telegram — изменения уже действуют.

Переустанавливать Python, venv, .env и systemd не нужно. Новый код подхватывается при перезапуске сервиса.

### Как загрузить изменения на сервер

**Вариант 1 — заново залить весь проект (проще всего):**

На **ПК** (PowerShell, папка `d:\VPN BOT`):

```
cd "d:\VPN BOT"
tar --exclude=venv --exclude=__pycache__ --exclude=*.db --exclude=.git -czvf vpn-bot.tar.gz .
scp vpn-bot.tar.gz root@155.212.164.135:/opt/
```

На **сервере**:

```
cd /opt/vpn-bot
tar -xzvf /opt/vpn-bot.tar.gz -C /opt/vpn-bot
systemctl restart vpn-bot support-bot
```

Так вы перезаписываете весь код в `/opt/vpn-bot`, но **не** трогаете папку `venv` и файл `.env` (их в архиве нет). После распаковки просто перезапускаете ботов.

**Вариант 2 — залить только отдельные файлы:**

Если изменили, например, один файл в `bot/handlers/`, можно скопировать только его. На **ПК**:

```
scp "d:\VPN BOT\bot\handlers\main.py" root@155.212.164.135:/opt/vpn-bot/bot/handlers/
```

Потом на сервере: `systemctl restart vpn-bot`. Аналогично для любого другого файла или папки.

### Если меняли только Mini App (webapp)

Мини-приложение у вас на Vercel (https://bitvpn.vercel.app). Обновления там делаются через деплой в Vercel (git push или загрузка в их интерфейсе). Сервер перезапускать не нужно — бот на сервере только отдаёт ссылку на этот URL. Перезапуск нужен только если вы меняли код самого бота (Python) или .env на сервере.

### Кратко

| Вопрос | Ответ |
|--------|--------|
| Нужно ли каждый раз разворачивать всё с нуля? | Нет. |
| Нужно ли перезагружать сервер? | Нет. Достаточно `systemctl restart vpn-bot` и/или `support-bot`. |
| Сколько ждать после перезапуска? | 1–2 секунды, потом уже можно тестировать в Telegram. |
| Нужно ли заново создавать .env и venv? | Нет, только если вы специально их удалили или меняли версию Python. |

## Готово. Полезные команды на будущее

Все команды ниже выполняются **на сервере** (в SSH).

| Что сделать | Команда |
|-------------|---------|
| Посмотреть логи VPN-бота в реальном времени | `journalctl -u vpn-bot -f` (выход: Ctrl+C) |
| Посмотреть логи бота поддержки | `journalctl -u support-bot -f` (выход: Ctrl+C) |
| Перезапустить оба бота | `systemctl restart vpn-bot support-bot` |
| Остановить оба бота | `systemctl stop vpn-bot support-bot` |
| Запустить снова | `systemctl start vpn-bot support-bot` |

После перезагрузки сервера боты запустятся сами.

## Если что-то пошло не так

- **В шаге 6 в `/opt/vpn-bot` нет run.py и папки bot** — архив собран не из той папки. На ПК убедитесь, что вы в `d:\VPN BOT` (там есть run.py и папка bot), и повторите шаги 4 и 5.
- **Бот не запускается (не running)** — выполните `journalctl -u vpn-bot -n 30` или `journalctl -u support-bot -n 30` и посмотрите последние строки. Часто причина — нет файла `.env` или неверные токены в нём (проверьте шаг 8).
- **Начать всё заново** — на сервере выполните: `systemctl stop vpn-bot support-bot`, затем `rm -rf /opt/vpn-bot /opt/vpn-bot.tar.gz`, и пройдите с шага 2 (или с шага 4 на ПК, затем 6–10 на сервере).
