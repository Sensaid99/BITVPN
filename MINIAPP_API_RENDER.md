# API мини-приложения на Render (без ngrok)

Чтобы реферальная ссылка и данные подписок подгружались в приложении, API должен быть доступен по **HTTPS**. Ngrok для этого не нужен — можно один раз развернуть API на **Render** (есть бесплатный тариф) и получить постоянный URL.

**Рекомендуется:** пошаговая инструкция с нуля (Blueprint + PostgreSQL) — в файле **RENDER_ПОШАГОВО.md**. Ниже — вариант через ручное создание сервиса.

---

## Шаг 1. Репозиторий на GitHub

Если проект ещё не в GitHub — создайте репозиторий и загрузите туда код (нужна папка с `api_miniapp.py`, `bot/`, `requirements.txt` и т.д., т.е. корень проекта).

---

## Шаг 2. Сервис на Render

1. Зайдите на **https://render.com** и войдите (можно через GitHub).
2. **Dashboard** → **New** → **Web Service**.
3. Подключите ваш **GitHub-репозиторий** (тот, где лежит VPN BOT с `api_miniapp.py`).
4. Настройки:
   - **Name:** например `bitvpn-miniapp-api`.
   - **Region:** любой.
   - **Branch:** `main` (или та ветка, где вы деплоите).
   - **Root Directory:** оставьте пустым (корень репозитория).
   - **Runtime:** **Python 3**.
   - **Build Command:**
     ```bash
     pip install -r requirements.txt
     ```
   - **Start Command:**
     ```bash
     uvicorn api_miniapp:app --host 0.0.0.0 --port $PORT
     ```
     (Render сам задаёт переменную `PORT`.)

5. **Environment** (переменные окружения) — добавьте те же, что нужны боту и API:
   - `BOT_TOKEN` — токен вашего VPN-бота.
   - `DATABASE_URL` — строка подключения к БД (как в `.env` у бота).
   - При необходимости остальные из вашего `.env` (например `BOT_USERNAME` и т.д.), если они используются в `api_miniapp` или в импортируемом коде.

6. Нажмите **Create Web Service**. Render соберёт проект и запустит API.

---

## Шаг 3. Взять URL и прописать в .env

1. После деплоя вверху страницы сервиса будет **URL**, например:
   `https://bitvpn-miniapp-api.onrender.com`
2. В **корне проекта** откройте **`.env`** и укажите этот адрес **без слэша в конце**:
   ```env
   MINIAPP_API_URL=https://bitvpn-miniapp-api.onrender.com
   ```
3. Сохраните файл и **перезапустите бота**.

После этого бот будет открывать мини-приложение с параметром `api=...`, приложение станет обращаться к вашему API на Render, и реферальная ссылка (и подписки/платежи) будут подгружаться. **Ngrok не нужен.**

---

## Если сборка падает: «Could not open requirements file: requirements.txt»

**Причина:** сборка запускается не из корня репозитория, либо файла нет в репозитории.

**Что сделать:**

1. **Root Directory в Render**  
   В сервисе: **Settings** → **Root Directory**. Должно быть **пусто**. Если там указана папка (например `bot` или `webapp`), очистите поле и сохраните — сборка должна идти из корня репо, где лежит `requirements.txt`.

2. **Проверьте GitHub**  
   Откройте репозиторий (например `https://github.com/Sensaid99/BITVPN`) и убедитесь, что в **корне** репозитория есть файл `requirements.txt` (рядом с `api_miniapp.py`, папкой `bot` и т.д.). Если его нет — добавьте его в корень проекта, закоммитьте и запушьте, затем заново задеплойте на Render (Manual Deploy или автодеплой по push).

3. **Перезапуск деплоя**  
   После исправления: в Render откройте сервис → **Manual Deploy** → **Deploy latest commit**.

---

## Важно

- На **бесплатном** тарифе Render после ~15 минут без запросов сервис «засыпает»; первый запрос после этого может идти 30–60 секунд. Для постоянной работы без задержек можно взять платный тариф или другой хостинг (VPS, Railway, Fly.io и т.п.).
- База данных должна быть доступна с серверов Render (например, облачная PostgreSQL). Локальная БД на вашем ПК с Render работать не будет — нужен доступный по сети адрес `DATABASE_URL`.
