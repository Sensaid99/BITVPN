■ ОДНА ССЫЛКА (bitvpn.vercel.app)

  Везде используется один проект Vercel:  https://bitvpn.vercel.app  (проект «bitvpn»).
  В .env уже прописано:
    WEBAPP_URL=https://bitvpn.vercel.app
    MINIAPP_API_URL=https://bitvpn.vercel.app

  Второй проект (bitvpn-eo9w) можно удалить: Vercel → Projects → bitvpn-eo9w → Settings → внизу «Delete Project».
  Убедитесь, что проект «bitvpn» подключён к репозиторию GitHub (Sensaid99/BITVPN), чтобы пуш вызывал деплой. На сервере в .env тоже пропишите WEBAPP_URL и MINIAPP_API_URL с https://bitvpn.vercel.app.

■ Если в Telegram открывается в браузере, а не внутри приложения

  • В .env бота (и на сервере) должен быть ровно один URL без слеша в конце:
      WEBAPP_URL=https://bitvpn.vercel.app
      MINIAPP_API_URL=https://bitvpn.vercel.app
  • По адресу https://bitvpn.vercel.app должна открываться страница приложения (не 500 и не 404). Если там ошибка — Telegram может открыть ссылку во внешнем браузере.

■ Если в Telegram «404: NOT FOUND» и Code: DEPLOYMENT_NOT_FOUND

  Это значит: по адресу WEBAPP_URL (bitvpn.vercel.app) на Vercel нет рабочего деплоя.
  1. Зайдите на https://vercel.com → войдите в аккаунт.
  2. Найдите проект «bitvpn». Если его нет — создайте: Add New → Project → Import репозитория Sensaid99/BITVPN (GitHub), Root Directory оставьте пустым, Deploy.
  3. Если проект есть: откройте его → вкладка Deployments. Если список пустой или последний деплой с ошибкой — нажмите «Redeploy» у последнего или сделайте новый пуш в GitHub (ДЕПЛОЙ_НА_СЕРВЕР.bat), чтобы запустился новый деплой.
  4. Дождитесь успешного деплоя (Status: Ready). Тогда https://bitvpn.vercel.app снова будет открываться и кнопка в боте заработает.
  • Кнопку «Открыть приложение» нажимать в самом Telegram (не переходить по ссылке из чата). Открытие именно через кнопку Web App запускает мини-апп внутри Telegram.
  • На сервере, где крутится бот, в .env обязательно те же WEBAPP_URL и MINIAPP_API_URL. Иначе кнопка отправит старый URL.

■ ШАГ 1. Залить изменения на GitHub (чтобы Vercel задеплоил)

  1.1. Откройте папку проекта в проводнике.
  1.2. Запустите файл:  ДЕПЛОЙ_ОДНА_КНОПКА.vbs
        (или дважды нажмите  ДЕПЛОЙ_НА_СЕРВЕР.bat)
  1.3. Когда попросит — введите пароль от сервера (1–2 раза).
  1.4. Дождитесь сообщения «Готово».

  Либо вручную в Cursor:
  — Нажмите на иконку ветки слева (Source Control) или Ctrl+Shift+G.
  — Нажмите «+» у изменённых файлов (Stage All).
  — Введите сообщение: fix miniapp
  — Нажмите ✓ Commit, затем кнопку Sync / Push.

  После пуша Vercel сам начнёт новый деплой (если проект привязан к репозиторию).

───────────────────────────────────────────────────────────────────

■ ШАГ 2. Настройки проекта на Vercel (по возможности)

  2.1. Зайдите на https://vercel.com и откройте проект (bitvpn).
  2.2. Вверху откройте вкладку  Settings → слева  General.
  2.3. Если видите пункт  Root Directory — убедитесь, что поле пустое (не указана папка webapp и т.п.), сохраните.
  2.4. Если пункта  Root Directory нет — ничего делать не нужно. В новых версиях Vercel его может не быть, проект уже использует корень репозитория.

───────────────────────────────────────────────────────────────────

■ ШАГ 3. Переменные окружения на Vercel (если мини-апп открывается, но нет тарифов/профиля)

  3.1. В том же проекте Vercel: Settings → Environment Variables.
  3.2. Должны быть заданы:
        BOT_TOKEN     — токен бота от BotFather
        DATABASE_URL  — строка подключения к PostgreSQL (Neon)
  3.3. Если их нет — нажмите Add, введите имя и значение, сохраните.
  3.4. После добавления/изменения переменных сделайте Redeploy последнего деплоя
        (Deployments → три точки у деплоя → Redeploy).

───────────────────────────────────────────────────────────────────

■ ШАГ 4. Открыть мини-апп после деплоя

  4.1. Дождитесь окончания деплоя на Vercel (зелёная галочка у последнего деплоя).
  4.2. В Telegram откройте бота @Bitvpnproxy_bot.
  4.3. Нажмите кнопку  «Открыть приложение ».

───────────────────────────────────────────────────────────────────

■ Когда будете менять интерфейс мини-апп (public/index.html)

  Перед пушем один раз запустите:  синхронизировать_миниапп_перед_деплоем.bat
  Он скопирует public\index.html в корень и в api\, чтобы после деплоя везде была одна версия.
