# Mini App на Vercel (без ngrok)

**Если при открытии приложения видите «404: NOT_FOUND»** — по указанному в `.env` адресу (`WEBAPP_URL`) ничего не развёрнуто. Нужно один раз задеплоить папку `webapp` на Vercel и прописать в `.env` **тот URL, который выдаст Vercel** (см. шаги ниже).

Vercel даёт постоянный HTTPS-адрес, не отваливается и не требует держать компьютер включённым.

---

## Шаг 1. Установите Node.js (если ещё нет)

- Скачайте LTS: https://nodejs.org/
- Установите, перезапустите терминал.

Проверка:
```bash
node -v
npm -v
```

---

## Шаг 2. Разверните Mini App на Vercel

**Вариант А — через сайт (проще):**

1. Зайдите на https://vercel.com и войдите (через GitHub или почту).
2. Нажмите **Add New… → Project**.
3. Если проект уже в GitHub — выберите репозиторий и папку **webapp** как Root Directory.  
   Если нет — выберите **Import Third-Party Git** или загрузите папку вручную (см. Вариант Б).
4. **Root Directory:** укажите `webapp` (или загрузите только содержимое папки `webapp`).
5. **Deploy**. После деплоя скопируйте URL вида `https://ваш-проект.vercel.app`.

**Вариант Б — через командную строку (рекомендуется):**

1. Запустите **`задеплоить_miniapp.bat`** (двойной клик). Откроется **отдельное окно CMD** — оно не закроется, в нём пройдёт вход в Vercel и деплой.  
   **Если окно всё равно закрывается:** откройте CMD вручную (Win+R → `cmd` → Enter), выполните:
   ```bat
   cd /d "d:\VPN BOT\webapp"
   npx vercel --prod
   ```
   Окно CMD останется открытым, можно спокойно нажать Y и дождаться браузера.

2. Если появится **«The specified token is not valid»** — в том же окне выполните:
   ```bat
   npx vercel login
   ```
   Откроется браузер для входа; после входа снова запустите `задеплоить_miniapp.bat` или в этой папке выполните `npx vercel --prod`.
3. На вопросы Vercel ответьте:
   - **Set up and deploy?** — Yes
   - **Which scope?** — ваш аккаунт
   - **Link to existing project?** — No
   - **Project name?** — введите **bitvpn** (тогда URL будет `https://bitvpn.vercel.app`) или любое имя
   - **Directory?** — текущая папка (`.`), Enter

4. В конце появится строка вида `Production: https://bitvpn.vercel.app` (или другой URL).  
   **Именно этот адрес** скопируйте в `.env` как `WEBAPP_URL=...`.

---

## Шаг 3. Подставьте URL в бота

1. Откройте файл **`.env`** в корне проекта (`d:\VPN BOT\.env`).
2. Найдите строку `WEBAPP_URL=...`.
3. Замените на ваш адрес Vercel (без слэша в конце):

```env
WEBAPP_URL=https://ваш-проект.vercel.app
```

4. Сохраните файл и перезапустите бота.

---

## Шаг 4. Обновления Mini App

После изменений в папке `webapp/` снова выполните из папки `webapp`:

```bash
npx vercel --prod
```

Или через сайт Vercel: заново задеплойте проект (Redeploy).

---

## Итог

| Раньше (ngrok)              | Сейчас (Vercel)              |
|-----------------------------|-----------------------------|
| Нужно держать ПК и ngrok   | Ничего не нужно держать     |
| URL меняется при перезапуске | Один постоянный URL         |
| Часто отваливается          | Стабильный хостинг          |

Кнопка **«📱 Открыть приложение»** в боте будет вести на этот постоянный адрес.

## Реферальная ссылка в приложении (почему «Загрузка…» и что сделать)

Чтобы в разделе **Профиль → Пригласить друга** сразу отображалась **ваша реферальная ссылка**, мини-приложение должно получать данные пользователя с вашего API.

**Что сделать с вашей стороны:**

1. **Разверните API мини-приложения по HTTPS.** Ngrok для этого не нужен — удобнее один раз задеплоить API на **Render** (бесплатный тариф) и получить постоянный URL. Пошагово: **`MINIAPP_API_RENDER.md`**. Альтернативы: Fly.io, ваш VPS или любой хостинг с Python и доступом к вашей БД.

2. **В файле `.env` в корне проекта** укажите адрес этого API:
   ```env
   MINIAPP_API_URL=https://ваш-api.onrender.com
   ```
   (без слэша в конце; подставьте свой URL, где крутится `api_miniapp`).

3. **Перезапустите бота.**  
   После этого бот будет открывать мини-приложение с параметром `api=...`. Приложение запросит у API данные пользователя (в т.ч. `referral_code`) и подставит реферальную ссылку.

4. **Открывайте приложение из меню бота** (кнопка «📱 Открыть приложение»).  
   Реферальная ссылка подгружается при старте приложения; если всё настроено, в шторке «Пригласить друга» сразу будет ваша ссылка.

**Если не настроено:** при открытии шторки будет подсказка, что нужно указать `MINIAPP_API_URL` в `.env` и открывать приложение из бота.
