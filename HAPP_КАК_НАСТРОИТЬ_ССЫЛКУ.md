# Как настроить Happ, чтобы сразу выдавалась ссылка после оплаты

После оплаты бот **сам** создаёт лимитированную ссылку через API Happ и отправляет её пользователю (сообщение + файл .txt). Чтобы это работало, нужно один раз получить ключи в кабинете Happ и прописать их в `.env`.

---

## Что происходит «сразу»

1. Пользователь оплачивает подписку в мини-приложении (СБП или карта).
2. ЮKassa подтверждает платёж (вебхук или кнопка «Проверить платёж» в боте).
3. Бот вызывает API Happ: «создать лимитированную ссылку» с лимитом устройств по тарифу (1, 3, 5 или 10).
4. Happ возвращает код (`install_code`), бот склеивает ссылку: **ваш базовый URL** + `?installid=` + код.
5. Бот **сразу** отправляет пользователю:
   - сообщение с текстом ссылки;
   - файл .txt со ссылкой (удобно скопировать и вставить в приложение Happ);
   - затем сообщение «Настроить VPN» с кнопками выбора устройства.

Никаких ручных действий — ссылка выдаётся автоматически после успешной оплаты.

---

## Пошаговая настройка

### Шаг 1. Аккаунт провайдера на Happ

1. Зайдите на **[happ-proxy.com](https://happ-proxy.com)** (или [happ.su](https://www.happ.su/)).
2. Зарегистрируйтесь / войдите как **провайдер** (не как обычный пользователь приложения).
3. В личном кабинете найдите:
   - **provider_code** — 8 символов (латиница/цифры);
   - **auth_key** — ключ для API (обычно 32 символа).

Если раздела «API» или «Для разработчиков» нет — напишите в поддержку Happ, что нужен доступ к API для лимитированных ссылок.

Документация API: [docs.happ-proxy.com](https://docs.happ-proxy.com/ru/getting-started/api).

---

### Шаг 2. Базовая ссылка подписки (HAPP_SUBSCRIPTION_URL)

Бот к этой ссылке добавляет параметр `installid=КОД`. Пользователь в итоге открывает, например:

`https://ваш-сервер.example.com/sub?installid=ABCD123456`

Вам нужна **базовая** ссылка (без `installid`):

- Если подписку отдаёт сам Happ по вашим настройкам — в кабинете провайдера будет ваш URL подписки, его и укажите.
- Если подписку отдаёт ваш сервер или другая панель — укажите тот URL, который отдаёт конфиг подписки в формате, поддерживаемом Happ.

Примеры: `https://sub.yourvpn.com/me`, `https://ваш-домен.com/sub` — **без** параметров в конце.

---

### Шаг 3. Привязка домена подписки (если требуется по документации Happ)

По [документации](https://docs.happ-proxy.com/ru/getting-started/api) для работы лимитированных ссылок домен подписки может потребоваться привязать к вашему provider через API:

- Эндпоинт: `GET /api/add-domain`
- Параметры: `provider_code`, `auth_key`, `domain_hash` (SHA-256 от домена в нижнем регистре, 64 hex-символа).

Сделать это нужно один раз (скриптом или вручную через curl), см. примеры в документации Happ.

---

### Шаг 4. Переменные в .env

На ПК (в папке проекта) и **на сервере** (где крутится бот) в файле `.env` добавьте или отредактируйте строки (без кавычек, без пробелов вокруг `=`):

```env
HAPP_API_URL=https://happ-proxy.com
HAPP_PROVIDER_CODE=ваш_8_символов
HAPP_AUTH_KEY=ваш_32_символа
HAPP_SUBSCRIPTION_URL=https://ваша-базовая-ссылка-подписки
```

- **HAPP_API_URL** — оставьте `https://happ-proxy.com`, если вам не выдали другой URL API.
- **HAPP_PROVIDER_CODE** — ваш provider_code из кабинета (8 символов).
- **HAPP_AUTH_KEY** — ваш auth_key из кабинета (32 символа).
- **HAPP_SUBSCRIPTION_URL** — базовая ссылка подписки (то, что вы раздаёте; бот сам добавит `?installid=...`).

Сохраните файл.

---

### Шаг 5. Перезапуск бота

На сервере после правки `.env`:

```bash
sudo systemctl restart vpn-bot
```

На ПК при локальном запуске — просто перезапустите бота.

---

## Проверка

1. Сделайте тестовую оплату (или проверку платежа в боте).
2. После успешного подтверждения оплаты пользователь должен **сразу** получить:
   - сообщение с текстом «Ваша ссылка для приложения Happ» и ссылкой;
   - файл .txt со ссылкой;
   - сообщение «Настроить VPN» с кнопками Android / iOS / Windows.

Если ссылка не приходит:

- Проверьте логи бота: `journalctl -u vpn-bot -n 50 --no-pager`. Ошибки вызова Happ API там логируются (например, «Happ API add-install failed»).
- Убедитесь, что в `.env` все четыре переменные заданы **без лишних пробелов** и точно совпадают с кабинетом Happ.
- Если Happ возвращает ошибку авторизации — перепроверьте **HAPP_PROVIDER_CODE** и **HAPP_AUTH_KEY**.

---

## Кратко

| Что нужно | Где взять |
|-----------|-----------|
| provider_code (8 символов) | Кабинет провайдера на happ-proxy.com |
| auth_key (32 символа) | Там же, раздел API |
| Базовая ссылка подписки | Ваш сервер подписки или URL из кабинета Happ |

Прописать в `.env` → перезапустить бота → после оплаты ссылка будет выдаваться автоматически.
