# Защита ссылки подписки: чтобы нельзя было «обойти» лимит

## Проблема

Если пользователь получает ссылку вида:

`https://95.181.175.67:2096/sub_bitvpn/bgmdn1s016p08yfb?installid=A8u9D1HjG3OA`

он может **удалить** `?installid=A8u9D1HjG3OA` и использовать только:

`https://95.181.175.67:2096/sub_bitvpn/bgmdn1s016p08yfb`

Такая ссылка без `installid` может всё ещё отдавать подписку с вашего сервера, то есть пользователь получает доступ без учёта лимита устройств Happ.

---

## Решение: редирект через ваш API

Бот может выдавать пользователю **не реальный URL подписки**, а ссылку на **ваш API**, которая делает редирект на реальную подписку с `installid`. Реальный URL (IP и путь) пользователь не видит; без кода подписка по редиректу не откроется.

### Как это работает

1. В `.env` на сервере (и в API) задаёте:
   ```env
   HAPP_SUBSCRIPTION_REDIRECT_BASE=https://ваш-домен-api
   ```
   Например: `https://nikolay.lisobyk.fvds.ru` (без слэша в конце).

2. Пользователь получает ссылку вида:
   ```text
   https://ваш-домен-api/sub/A8u9D1HjG3OA
   ```
   Вместо `https://95.181.175.67:2096/sub_bitvpn/bgmdn1s016p08yfb?installid=A8u9D1HjG3OA`.

3. Когда пользователь (или приложение Happ) открывает `https://ваш-домен-api/sub/A8u9D1HjG3OA`, ваш API отвечает **редиректом 302** на реальный URL с `installid`. Подписку по-прежнему отдаёт ваш сервер подписок (95.181.175.67), API только подставляет правильный `installid`.

4. Если кто-то попытается открыть `https://ваш-домен-api/sub/` без кода или с неправильным кодом — API вернёт **404**. Реальный URL подписки из ссылки не виден.

### Что прописать

- **На API-сервере** в `.env`:
  ```env
  HAPP_SUBSCRIPTION_REDIRECT_BASE=https://nikolay.lisobyk.fvds.ru
  ```
  (подставьте свой домен API, тот же, что в `MINIAPP_API_URL` / `WEBAPP_URL`.)

- Перезапустите API (и бота, если он на том же сервере):
  ```bash
  sudo systemctl restart miniapp-api
  ```

После этого **новые** ссылки (после оплаты, «Мой конфиг», мини-апп) будут в формате `https://ваш-домен/sub/КОД`. Старые сохранённые ссылки с полным URL продолжат работать, но новые пользователи реальный URL не увидят.

### Проверка

- Откройте в браузере: `https://ваш-домен-api/sub/XXXXXXXXXXXX` (подставьте любой 12-символьный код, например из старой ссылки после `installid=`).
- Должен произойти редирект на ваш сервер подписок с `?installid=...`, и подписка должна подгрузиться (или показать конфиг).
- Откройте `https://ваш-домен-api/sub/` без кода — должен быть ответ **404**.
