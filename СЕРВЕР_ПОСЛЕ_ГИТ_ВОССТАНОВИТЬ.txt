  ПОСЛЕ НАСТРОЙКИ GIT НА СЕРВЕРЕ БОТ НЕ РАБОТАЕТ — ВОССТАНОВЛЕНИЕ
═══════════════════════════════════════════════════════════════════

При команде "git reset --hard" на сервере папка /opt/vpn-bot была заменена
кодом с GitHub. Файл .env в репозитории не хранится — он мог пропасть.
Без .env бот не запускается (нет BOT_TOKEN, DATABASE_URL и т.д.).

■ ВАРИАНТ 1. Самый простой — скопировать .env с ПК и перезапустить бота

  1. На ПК откройте папку проекта (d:\VPN BOT).
  2. Двойной клик по  СКОПИРОВАТЬ_ENV_НА_СЕРВЕР.bat  — скопируется полный .env с ПК на сервер.
  3. Введите пароль от сервера, когда попросит.
  4. По SSH на сервере выполните:  sudo systemctl restart vpn-bot
  5. Проверьте бота в Telegram: отправьте /start.

  Важно: команду scp нужно выполнять с ПК (этот .bat), а не с сервера. Иначе на сервер попадёт пустой или 31-байтный .env.

■ ВАРИАНТ 2. Вручную скопировать .env и перезапустить

  На ПК в папке проекта (где лежит .env) откройте CMD или PowerShell и выполните:

    scp .env root@155.212.164.135:/opt/vpn-bot/.env

  Введите пароль от сервера. Затем подключитесь к серверу и перезапустите бота:

    ssh root@155.212.164.135
    sudo systemctl restart vpn-bot
    exit

■ Если кнопка «Открыть приложение» даёт 404 (NOT FOUND)

  Это из-за того, что на сервере в .env нет или неверный WEBAPP_URL.
  Подробно и по шагам: откройте файл  ОШИБКА_404_ПОСЛЕ_ГИТ.txt

■ Если бот всё равно не запускается (status=1/FAILURE)

  1. Посмотрите ошибку в логах (на сервере по SSH):

       sudo journalctl -u vpn-bot -n 80 --no-pager

     В конце будет трейсбек Python или сообщение об ошибке — по нему можно понять причину.

  2. Проверьте .env на сервере (часто он оказывается пустым или 31 байт — это неполный файл):

       ls -la /opt/vpn-bot/.env
       wc -c /opt/vpn-bot/.env

     Размер должен быть несколько сотен байт (BOT_TOKEN, DATABASE_URL, WEBAPP_URL и т.д.). Если 31 или 0 — копировать нужно **с вашего ПК**, а не с другого сервера. На ПК откройте CMD в папке **d:\VPN BOT** и выполните:

       scp .env root@155.212.164.135:/opt/vpn-bot/.env

     (Подставьте свой IP, если другой. Команду выполнять на ПК, не на сервере.)

  3. Проверьте, что venv на месте (после git reset он не удаляется, но бывает сбит путь):

       ls -la /opt/vpn-bot/venv/bin/python

     Если файла нет — на сервере создайте venv заново:
       cd /opt/vpn-bot
       python3 -m venv venv
       ./venv/bin/pip install -r requirements.txt
     Затем: sudo systemctl restart vpn-bot
