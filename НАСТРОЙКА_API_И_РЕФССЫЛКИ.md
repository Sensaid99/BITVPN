# Настройка API и реферальной ссылки в мини-приложении

Если при открытии реферальной шторки показывается «Ссылка появится после загрузки данных» или «Откройте приложение из меню бота…» — нужно настроить API и переменные в `.env`.

---

## Что за что отвечает

| Что | Где | Зачем |
|-----|-----|--------|
| **WEBAPP_URL** | .env | Адрес, по которому открывается мини-приложение. Если API на Render — укажите тот же URL, что и MINIAPP_API_URL (приложение и API на одном сервере). |
| **MINIAPP_API_URL** | .env | Адрес API, откуда приложение берёт рефссылку, подписки, профиль. На Render — URL вашего сервиса bitvpn-api. |
| **BOT_USERNAME** | .env | Username бота без `@` (например `Bitvpnproxy_bot`). Нужен, чтобы рефссылка имела вид `https://t.me/Bitvpnproxy_bot?start=...` |

Без **MINIAPP_API_URL** приложение не знает, куда слать запросы — поэтому рефссылка не подгружается.

---

## Вариант 1: API на Render (постоянный HTTPS)

Подробно шаги расписаны в **MINIAPP_API_RENDER.md**. Кратко:

### 1. Репозиторий на GitHub

Код проекта (с `api_miniapp.py`, `bot/`, `requirements.txt` в корне) должен быть в GitHub.

### 2. Сервис на Render

1. Зайти на [render.com](https://render.com), войти (можно через GitHub).
2. **New** → **Web Service**.
3. Подключить ваш репозиторий.
4. Настройки:
   - **Name:** например `bitvpn-api`.
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `uvicorn api_miniapp:app --host 0.0.0.0 --port $PORT`
5. **Environment** — добавить переменные (как в вашем `.env`):
   - `BOT_TOKEN` — токен VPN-бота.
   - `DATABASE_URL` — **обязательно база, доступная из интернета** (например PostgreSQL на Render или ElephantSQL).  
     Локальный файл `sqlite:///vpn_bot.db` на Render работать не будет: у сервера Render нет доступа к вашему диску.
6. Создать сервис. Дождаться деплоя и скопировать **URL** (например `https://bitvpn-api.onrender.com`).

### 3. .env у бота (на вашем ПК)

В корне проекта в файле **`.env`** указать (без слэша в конце):

```env
MINIAPP_API_URL=https://bitvpn-api.onrender.com
BOT_USERNAME=Bitvpnproxy_bot
```

Сохранить файл и **перезапустить бота**.

### 4. Как открывать приложение

Приложение нужно открывать **из бота** (кнопка «Открыть приложение»). Тогда бот подставит в ссылку параметр `api=...`, и приложение начнёт обращаться к вашему API. Если открыть приложение напрямую по адресу Vercel без бота — параметра `api` не будет, и рефссылка не загрузится.

## Вариант 2: API локально + ngrok (для проверки)

Если пока не хотите настраивать Render и облачную БД:

1. Запустить API локально: двойной клик по **ЗАПУСТИТЬ_МИНИАПП_API.bat** (поднимется на `http://localhost:8765`).
2. Запустить ngrok: `ngrok http 8765`.
3. Скопировать **https**-адрес из ngrok (например `https://abc123.ngrok.io`).
4. В **`.env`** прописать:
   ```env
   MINIAPP_API_URL=https://abc123.ngrok.io
   BOT_USERNAME=Bitvpnproxy_bot
   ```
5. Перезапустить бота и открывать мини-приложение из бота.

Тогда API и БД остаются на вашем ПК, а в интернет API выходит через ngrok.

---

## Проверка

- В боте нажать «Открыть приложение».
- Зайти в раздел **Рефералы** и открыть шторку с реферальной ссылкой.
- Если всё настроено, вместо «Загрузка…» появится ссылка вида `https://t.me/Bitvpnproxy_bot?start=...`, и её можно копировать.

Если ссылка по-прежнему не появляется:

1. Убедиться, что в `.env` указаны **MINIAPP_API_URL** и **BOT_USERNAME** и бот перезапущен после изменений.
2. Открывать приложение **именно из бота**, а не по прямой ссылке в браузере.
3. Для Render — убедиться, что сервис задеплоен и **DATABASE_URL** на Render указывает на облачную БД (не на локальный файл).
