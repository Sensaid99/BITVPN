# Интеграция с Happ (Happ-Proxy)

Happ — приложение и панель для прокси/VPN с поддержкой лимита устройств. Через API вы создаёте **лимитированную ссылку** (код установки); пользователь добавляет её в приложение Happ и получает доступ на N устройств.

Документация: [docs.happ-proxy.com](https://docs.happ-proxy.com/ru/getting-started/api), [happ.su](https://www.happ.su/main/ru/dev-docs).

---

## 1. Где взять Happ и ключи

1. Зарегистрируйтесь на **[happ-proxy.com](https://happ-proxy.com)**.
2. В личном кабинете (профиль или верхний правый угол) найдите:
   - **provider_code** — 8 символов (A–Z, a–z, 0–9)
   - **auth_key** — 32 символа (ключ авторизации для API)
3. Подготовьте **URL подписки** — адрес, по которому отдаётся ваша подписка (список серверов). Его вы настраиваете в панели Happ или на своём сервере. Формат: обычный URL подписки (например `https://ваш-домен.com/sub`). К этому URL бот будет добавлять параметр `installid=КОД`, чтобы ограничить число устройств.

---

## 2. Что прописать в .env

В корне проекта в файле **.env** добавьте (или раскомментируйте):

```env
# Happ (Happ-Proxy) — выдача лимитированных ссылок вместо WireGuard-конфига
HAPP_API_URL=https://happ-proxy.com
HAPP_PROVIDER_CODE=Ваш8Симв
HAPP_AUTH_KEY=ваш_32_символьный_ключ_авторизации
HAPP_SUBSCRIPTION_URL=https://ваш-домен.com/sub
```

- **HAPP_PROVIDER_CODE** и **HAPP_AUTH_KEY** — из профиля на happ-proxy.com.
- **HAPP_SUBSCRIPTION_URL** — базовый URL подписки (без параметров). К нему бот добавит `?installid=КОД` или `&installid=КОД`. Пользователь получит эту ссылку и вставит её в приложение Happ.

Если эти переменные **не заданы**, бот продолжит работать по старой схеме (демо WireGuard-конфиг).

---

## 3. Как выдаются ссылки (ничего вручную делать не нужно)

Бот **сам выдаёт** ссылки Happ в двух случаях:

| Когда | Что получает пользователь |
|-------|---------------------------|
| **После успешной оплаты** | Сообщение с текстом ссылки + файл `.txt` со ссылкой (удобно вставить в Happ). |
| **«Мой конфиг» / «Настроить на этом устройстве»** | Та же ссылка ещё раз (сообщение + файл `.txt` + QR). |

Как это устроено внутри:
- Бот вызывает API Happ: **создать лимитированную ссылку** (`install_limit` = число устройств по тарифу: 1, 3, 5 или 10).
- В ответ приходит **install_code**. Бот формирует ссылку: `HAPP_SUBSCRIPTION_URL` + `?installid=` + `install_code` (или `&installid=`, если в URL уже есть `?`).
- Пользователю отправляется эта ссылка и файл с ней. Конфиг WireGuard в этом режиме не генерируется.
- Пользователь открывает ссылку или вставляет её в приложение Happ и подключается. Лимит устройств контролирует Happ.

---

## 4. Важно

- **URL подписки** должен быть доступен по HTTPS и отдавать подписку в формате, который умеет Happ (обычная или зашифрованная подписка). Настраивается отдельно (ваш сервер или панель Happ).
- Количество устройств берётся из тарифа (например `1_month_3` → 3 устройства). Если в тарифе число устройств не указано, используется 1.
- Подробное описание API: [Создать лимитированную ссылку](https://docs.happ-proxy.com/ru/getting-started/api#создать-лимитированную-ссылку), [Лимитированные ссылки](https://www.happ.su/main/ru/dev-docs/limitirovannye-ssylki).
