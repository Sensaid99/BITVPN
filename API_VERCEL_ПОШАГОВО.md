# API мини-приложения на Vercel (без карты)

Чтобы и приложение, и API были на Vercel — один аккаунт, без Render и без привязки карты.

---

## Что уже сделано в проекте

- В корне добавлены папка **api/** с точкой входа и **vercel.json** — API готов к деплою на Vercel.
- Нужна **база данных**: бесплатный PostgreSQL (например Neon) — карта не нужна.

---

## Шаг 1. Бесплатная база PostgreSQL (Neon)

1. Зайдите на [neon.tech](https://neon.tech) и войдите (через GitHub удобно).
2. Создайте проект (Create Project), выберите регион.
3. После создания скопируйте **Connection string** (строка вида `postgresql://user:password@ep-xxx.region.aws.neon.tech/neondb?sslmode=require`). Это ваш **DATABASE_URL**.

---

## Шаг 2. Создать таблицы в базе (один раз)

Чтобы API видел пользователей и подписки, таблицы должны существовать в Neon.

**Вариант А.** На своём ПК в **.env** пропишите скопированный DATABASE_URL от Neon:

```env
DATABASE_URL=postgresql://...от Neon...
```

Запустите бота один раз (**ЗАПУСТИТЬ_БОТА.bat**) и отправьте боту в Telegram команду `/start`. Бот создаст таблицы в Neon. После этого можно снова поставить в .env свою старую базу или оставить Neon — как вам удобно.

**Вариант Б.** Либо выполните SQL-схему вручную в Neon (Console → SQL Editor) — схема таблиц есть в коде бота (`bot/models/database.py`). Обычно проще вариант А.

---

## Шаг 3. Новый проект на Vercel для API

1. Зайдите на [vercel.com](https://vercel.com), войдите в тот же аккаунт, что и для мини-приложения.
2. Нажмите **Add New** → **Project**.
3. Импортируйте репозиторий **Sensaid99/BITVPN** (тот же, что и для приложения).
4. **Важно:** в настройках проекта:
   - **Root Directory** обязательно оставьте **пустым** (не указывайте `webapp` и никакую подпапку). Иначе Vercel не увидит `app.py`, `index.py`, `api_miniapp.py` и выдаст «No fastapi entrypoint found». Для API нужен корень репозитория, где лежат `app.py`, `index.py`, `api_miniapp.py`, `api/`, `bot/`, `vercel.json`.
   - **Framework Preset** можно оставить Other или не трогать.
   - **Build Command** — можно оставить пустым (Vercel сам подтянет зависимости для Python).
5. В блоке **Environment Variables** добавьте переменные (обязательно до деплоя):

   | Name           | Value                    |
   |----------------|--------------------------|
   | `BOT_TOKEN`    | Токен вашего VPN-бота от BotFather |
   | `DATABASE_URL` | Строка подключения от Neon (из шага 1) |

6. Нажмите **Deploy**.

---

## Шаг 4. Узнать URL API

После деплоя Vercel покажет адрес проекта, например:

`https://bitvpn-api-xxx.vercel.app`

Скопируйте его **без слеша в конце** — это адрес вашего API.

---

## Шаг 5. Прописать URL в .env бота

На ПК в папке проекта откройте **.env** и задайте:

```env
WEBAPP_URL=https://ваш-мини-апп.vercel.app
MINIAPP_API_URL=https://bitvpn-api-xxx.vercel.app
BOT_USERNAME=Bitvpnproxy_bot
```

- **WEBAPP_URL** — адрес мини-приложения на Vercel (тот, что получили из «задеплоить_miniapp.bat»).
- **MINIAPP_API_URL** — адрес только что задеплоенного API (шаг 4).
- **BOT_USERNAME** — username бота без @.

Сохраните файл и **перезапустите бота**.

---

## Шаг 6. Одна база для бота и API (рекомендуется)

Чтобы в мини-приложении отображались те же пользователи и подписки, что и в боте, и бот, и API должны использовать **один и тот же DATABASE_URL**.

- В **Vercel** (Environment Variables) уже указан DATABASE_URL от Neon.
- На **ПК** в **.env** тоже пропишите тот же **DATABASE_URL** (от Neon) и перезапустите бота. Тогда бот и API будут работать с одной базой.

---

## Проверка

1. В браузере откройте `https://ваш-api.vercel.app/` — должна открыться **страница мини-приложения** (HTML). Если видите JSON — в репозитории должна быть папка **public** с файлом **public/index.html** (копия из webapp); задеплойте заново.
2. В Telegram откройте бота → **Открыть приложение** → должно открыться приложение (не «Загрузки»). Если правки были в webapp — запустите **sync_webapp_to_public.bat**, затем push и redeploy.

---

## Если что-то не работает

- **«No fastapi entrypoint found»:**  
  1. В логе сборки посмотрите **Commit:** — если это старый коммит (не последний на main), то Vercel собрал старую версию без `app.py`. Не нажимайте просто **Redeploy** (он пересобирает тот же коммит). Сделайте новый push в `main` (например через **запушить_на_гитхаб_и_редиплой.bat**) — тогда запустится новый деплой с последним кодом.  
  2. В проекте Vercel: **Settings** → **General** → **Root Directory** должно быть **пустым**. Если там указана папка (например `webapp`) — очистите и сохраните, затем задеплойте заново.

- **Ошибка при деплое:** проверьте, что Root Directory пустой и в репозитории в корне есть `app.py`, `index.py`, `api_miniapp.py`, `api/`, `bot/`, `vercel.json`, `pyproject.toml`, `requirements.txt`.
- **В приложении нет данных:** проверьте, что в Vercel в Environment Variables заданы **BOT_TOKEN** и **DATABASE_URL**, и что бот на ПК использует тот же DATABASE_URL (если хотите общую базу).
- **Реферальная ссылка не появляется:** в .env должен быть указан **BOT_USERNAME** (без @) и **MINIAPP_API_URL** на ваш API на Vercel.

- **«This Serverless Function has crashed» / 500:** откройте в Vercel проект API → вкладка **Logs** (или **Runtime Logs**). Там будет текст ошибки (например, нет модуля, ошибка БД). Убедитесь, что в **Settings** → **Environment Variables** заданы **BOT_TOKEN** и **DATABASE_URL** (строка от Neon). Без них API может падать при первом запросе.

- **«This Serverless Function has crashed» / 500:**  
  1. В Vercel откройте проект API → **Deployments** → выберите последний деплой → **Functions** → откройте функцию и посмотрите **Logs** (Runtime Logs). Там будет причина падения.  
  2. В **Settings** → **Environment Variables** обязательно заданы **BOT_TOKEN** (токен бота от BotFather) и **DATABASE_URL** (строка от Neon). Без них загрузка БД не удаётся.  
  3. Сделайте **новый push в main** и дождитесь деплоя (не только Redeploy старого коммита). В коде API включена ленивая загрузка БД — маршруты GET / и GET /health не требуют БД и не должны падать.

- **В Telegram открываются «Загрузки» вместо приложения:** мини-приложение должно раздаваться как статический HTML. В репозитории должна быть папка **public** с файлом **public/index.html**. Убедитесь, что при push в репозитории есть эта папка и файл, затем сделайте новый деплой на Vercel (не Redeploy старого коммита — именно push + деплой). В .env у бота **WEBAPP_URL** должен быть `https://ваш-api.vercel.app` (без слеша в конце).
