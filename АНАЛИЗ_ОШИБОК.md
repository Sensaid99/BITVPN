# Анализ ошибок бота (22.02.2026)

## Найденные проблемы

### 1. **SQLAlchemy: User не привязан к сессии (главная причина «не реагирует»)**
- **Ошибка в логе:** `Instance <User at 0x...> is not bound to a Session; attribute refresh operation cannot proceed`
- **Причина:** `get_or_create_user()` закрывал сессию (`session.close()`) и возвращал объект `User`. Потом в обработчиках вызывалось `user.has_active_subscription` → обращение к `user.subscriptions` → ленивая загрузка из БД при уже закрытой сессии → падение.
- **Исправление:** В `bot/handlers/main.py` в `get_or_create_user()` перед `return` добавлены `session.refresh(user)` и принудительная загрузка связи: `_ = list(user.subscriptions)`. После этого объект можно безопасно использовать после закрытия сессии.

### 2. **Конфликт двух экземпляров бота (409 Conflict)**
- **Ошибка в логе:** `Conflict: terminated by other getUpdates request; make sure that only one bot instance is running`
- **Причина:** Запущено несколько процессов `python run.py` (разные окна, Cursor + батник и т.д.). У одного токена может быть только один long polling.
- **Исправление:** В `ЗАПУСТИТЬ_БОТА.bat` перед запуском добавлено завершение всех процессов `python.exe` и пауза 2 сек. Запускать бота только одним способом.

### 3. **Старая БД (no such column: users.telegram_id)**
- **Ошибка в логе:** `sqlite3.OperationalError: no such column: users.telegram_id`
- **Причина:** Использовалась БД от старого бота с другой схемой.
- **Исправление:** Переход на файл `vpn_bot.db` (в `.env`: `DATABASE_URL=sqlite:///vpn_bot.db`). В `bot/models/database.py` при старте проверяется схема; при несовпадении таблицы пересоздаются. В `run.py` при запуске удаляется старый файл `vpn_users.db`, если он есть.

### 4. **Обработчик рассылки перехватывал все сообщения**
- **Проблема:** MessageHandler для рассылки был без фильтра и вызывался на любое текстовое сообщение; у обычных пользователей просто делал `return`, из-за чего создавалось впечатление «ничего не работает».
- **Исправление:** Добавлен фильтр `filters.User(user_id=Config.ADMIN_IDS)` — обработчик рассылки срабатывает только для админов.

### 5. **Порядок обработчиков кнопок**
- **Проблема:** ConversationHandler (покупка VPN) был зарегистрирован раньше обработчиков кнопок «Профиль», «Помощь» и т.д., из-за чего часть callback_query могла обрабатываться не теми хендлерами.
- **Исправление:** Обработчики кнопок меню (profile, help, support, referral, main_menu) зарегистрированы в `bot/main.py` **до** ConversationHandler.

---

## Как запускать после исправлений

1. Запускать бота **только одним способом**: двойной клик по **`ЗАПУСТИТЬ_БОТА.bat`** (он сам остановит старые процессы и запустит один экземпляр).
2. Не запускать одновременно из Cursor (Run) и из батника.
3. При появлении ошибок смотреть файл в папке `logs/` (например `vpn_bot_YYYYMMDD.log`).
