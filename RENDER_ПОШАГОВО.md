# Пошаговый деплой API на Render (с нуля)

Чтобы реферальная ссылка и данные в мини-приложении работали, API должен быть в интернете по HTTPS. Ниже — пошаговая настройка на Render.

---

## Шаг 1. Репозиторий на GitHub

1. Создайте репозиторий на [github.com](https://github.com) (если ещё нет).
2. В корне проекта должны быть файлы:
   - `api_miniapp.py`
   - `bot/` (папка с ботом)
   - `requirements.txt`
   - `render.yaml` (уже есть в проекте)
3. Закоммитьте и запушьте всё в GitHub (включая `render.yaml`).

---

## Шаг 2. Render: Blueprint (база + API одним деплоем)

1. Зайдите на [render.com](https://render.com) и войдите (через GitHub удобнее).
2. В Dashboard нажмите **New** → **Blueprint**.
3. Подключите ваш **GitHub-репозиторий** (тот, где лежит VPN BOT).
4. Render подхватит файл **render.yaml** из корня и покажет:
   - базу данных **bitvpn-db** (PostgreSQL, бесплатно);
   - сервис **bitvpn-api** (Web Service).
5. Нажмите **Apply**.
6. Если на GitHub код лежит внутри папки **VPN BOT**: зайдите в сервис **bitvpn-api** → **Settings** → **Root Directory** укажите `VPN BOT` → **Save**. Затем **Manual Deploy** → **Deploy latest commit**.

---

## Шаг 3. Задать BOT_TOKEN для API

1. После создания зайдите в сервис **bitvpn-api** (Web Service).
2. Откройте вкладку **Environment**.
3. Добавьте переменную:
   - **Key:** `BOT_TOKEN`
   - **Value:** токен вашего VPN-бота (как в `.env` на ПК).
4. Сохраните (**Save Changes**).

Без `BOT_TOKEN` API не сможет проверять данные от Telegram — рефссылка не появится.

---

## Шаг 4. Дождаться деплоя

1. На вкладке **Events** / **Logs** дождитесь успешного деплоя (Build и Deploy без ошибок).
2. Вверху страницы сервиса будет **URL**, например:
   `https://bitvpn-api.onrender.com`  
   Скопируйте его — это адрес вашего API.

Если сборка падает с ошибкой «Could not open requirements file» — проверьте, что в **корне** репозитория на GitHub есть `requirements.txt` и `api_miniapp.py` (и что в Render выбран корень репозитория, а не подпапка).

---

## Шаг 5. Подключить бота к той же базе (чтобы рефссылки и пользователи совпадали)

API и бот должны работать с **одной и той же** базой. Сейчас API на Render уже использует PostgreSQL с Render. Бот на вашем ПК должен подключаться к этой же базе.

1. В Render откройте базу **bitvpn-db** (PostgreSQL).
2. Во вкладке **Info** найдите **External Database URL** (строка вида `postgresql://...`).  
   Она может называться **Connection string** или **PSQL Command** — скопируйте именно **внешний** URL (external), не internal.
3. На вашем ПК в папке проекта откройте файл **`.env`**.
4. Замените строку с базой на скопированный URL:
   ```env
   DATABASE_URL=postgresql://user:password@host/database?sslmode=require
   ```
   (значение подставьте то, что скопировали; у Render часто в конце добавляют `?sslmode=require` — оставьте как есть.)
5. Пропишите адрес API и username бота (без `@`):
   ```env
   MINIAPP_API_URL=https://bitvpn-api.onrender.com
   BOT_USERNAME=Bitvpnproxy_bot
   ```
   Вместо `bitvpn-api.onrender.com` подставьте **ваш** URL из шага 4.
6. Сохраните `.env`.

---

## Шаг 6. Перезапустить бота

1. Остановите бота на ПК (если он запущен).
2. Запустите снова.

При первом запуске с новой `DATABASE_URL` бот создаст таблицы в PostgreSQL на Render (если их ещё нет). Дальше и бот, и API будут использовать одну базу: пользователи, реферальные коды и подписки будут видны и в боте, и в мини-приложении.

---

## Шаг 7. Проверка

1. В Telegram откройте бота и нажмите **«Открыть приложение»** (открывать нужно именно из бота).
2. Перейдите в раздел **Рефералы** и откройте шторку с реферальной ссылкой.
3. Должна появиться ссылка вида `https://t.me/Bitvpnproxy_bot?start=...` (без «Загрузка…» и без сообщения про MINIAPP_API_URL).

Если ссылка не появляется:

- Убедитесь, что в Render у сервиса **bitvpn-api** задан **BOT_TOKEN** и деплой прошёл без ошибок.
- Проверьте, что в `.env` указаны **MINIAPP_API_URL** (без слэша в конце) и **BOT_USERNAME**, и бот перезапущен после изменений.
- Открывайте приложение только из бота, а не по прямой ссылке в браузере.

---

## Важно

- На бесплатном тарифе Render сервис «засыпает» после ~15 минут без запросов; первый запрос после этого может выполняться 30–60 секунд. Рефссылка тогда подгрузится с задержкой.
- База **bitvpn-db** и сервис **bitvpn-api** созданы одним Blueprint; при повторном деплое через тот же Blueprint они не дублируются.

После выполнения этих шагов API на Render настроен, бот и мини-приложение работают с одной базой, рефссылка и данные подписок должны отображаться корректно.

---

## Обновление кода (редиплой)

Работаете в **одной и той же папке** проекта. Чтобы отправить изменения на GitHub и запустить редиплой на Render:

1. Запустите файл **`запушить_на_гитхаб_и_редиплой.bat`** в корне проекта.
2. Скрипт сделает `git add`, `commit` и `push` в текущий репозиторий (ветка `main` или `master`).
3. Если Render подключён к этому репозиторию, он сам подхватит push и сделает редиплой — **новую папку или репозиторий создавать не нужно**.

Один раз нужно настроить:
- Если папка ещё не под Git: скрипт предложит выполнить `git init` и добавить remote:  
  `git remote add origin https://github.com/ВАШ_ЛОГИН/ВАШ_РЕПОЗИТОРИЙ.git`
- После этого достаточно нажимать `запушить_на_гитхаб_и_редиплой.bat` при любых изменениях в коде.
