Почему цены не изменились — проверьте по шагам

■ В БОТЕ (Telegram) всё ещё 299 или старые цены
  1. Полностью остановите бота (закройте окно/терминал или остановите ЗАПУСТИТЬ_БОТА.bat).
  2. Убедитесь, что в .env записано:
     PLAN_1_MONTH_PRICE=100
     PLAN_3_MONTH_PRICE=285
     PLAN_6_MONTH_PRICE=540
     PLAN_9_MONTH_PRICE=765
     PLAN_12_MONTH_PRICE=960
  3. Запустите бота снова. Откройте бота в Telegram → Купить VPN — должны быть новые цены.

■ В МИНИ-АПП всё ещё старые цены
  Мини-апп берёт цены с API на Vercel. На Vercel нет вашего .env — там свои переменные.
  1. Зайдите на vercel.com → ваш проект (API, например bitvpn-eo9w).
  2. Settings → Environment Variables.
  3. Добавьте или измените (значения как в .env):
     PLAN_1_MONTH_PRICE = 100
     PLAN_3_MONTH_PRICE = 285
     PLAN_6_MONTH_PRICE = 540
     PLAN_9_MONTH_PRICE = 765
     PLAN_12_MONTH_PRICE = 960
  4. Сохраните. Перейдите в Deployments → у последнего деплоя ⋮ (три точки) → Redeploy.
  5. Дождитесь окончания редеплоя (1–2 мин).
  6. В Telegram закройте мини-апп полностью и откройте снова (или закройте чат с ботом и зайдите заново) — приложение запросит тарифы с API заново.

■ Если в Vercel уже были старые значения (299, 799…)
  Пока вы не замените их на 100, 285… и не сделаете Redeploy, API будет отдавать старые цены. Просто изменить .env на ПК для мини-апп не достаточно — мини-апп смотрит на Vercel.
